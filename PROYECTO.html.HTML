1. ALGORITMO

secuencias finitas y ordenadas de instrucciones , pasos lógicos o reglas que se diseñan para resolver un problema específico realizar una tarea o alcanzar un objetivo determinado , estas instrucciones son claras y precisas y deben llevar a un resultado correcto.

2.LENGUAJE DE PROGRAMACIÓN

Conjunto de reglas y símbolos que permite al ser humano dar instrucciones a una computadora para que realice tareas específicas.

3.COMPILADOR

es un programa que traduce el código fuente en un lenguaje de programación de alto nivel ( comprendido por el hombre ) a otro lenguaje como código máquina o código de bytes , que la computadora puede entender y ejecutar directamente.

4.INTÉRPRETE

es un programa que ejecuta código fuente en línea por línea en tiempo de ejecución traducido a código máquina que la computadora pueda entender sin necesidad de una traducción de código previa.

5.SISTEM OPERATIVO

Programa o conjunto de programas que actúa como intermediario fundamental entre el hardware ( los componentes físicos de un dispositivo ) y las aplicaciones que el usuario desea ejecutar.

6.KERNEL O (NÚCLEO )

componente central de un sistema operativo que actúa como puente entre el hardware y el software gestionando los recursos del sistema como la memoria y la CPU para que las aplicaciones puedan funcionar.

7.BIOS/UEFI

Son el firmware que arranca la computadora actuando como puente entre el hardware y el sistema operativo.

8.DRIVER

Es un software que actúa como un traductor entre el sistema operativo y un componente hardware permitiendo que ambos se comuniquen y funcionen juntos.

9.FIRMWARE

Es un software integrado en un dispositivo de hardware que contiene las instrucciones necesarias para que ese hardware funcione correctamente actuando como un puente entre el software de nivel superior y el componente físico .

10.PROCESO

es un programa de ejecución que incluye las instrucciones del programa , su estado de ejecución actual , los registros de la CPU y la memoria de trabajo que ha reservado.

11.HILO(THREAD)

unidad más pequeña de ejecución que un sistema operativo puede gestionar, una secuencia de instrucciones dentro de un proceso que se ejecuta de forma concurrente o en paralelo con otros hilos.

12.MEMORIA RAM

Memoria de corto plazo de una computadora que almacena temporalmente los datos y programas que se están usando en el momento , su principal característica es la velocidad con la que el procesador puede acceder a la información .

13.ROM

Memoria temporal de alta velocidad que almacena datos de uso frecuente para acelerar el acceso del procesador de la información . Actúa como un intermediario entre la CPU y la memoria principal (memoria ram ) manteniendo

14.MEMORIA CACHE

Un tipo de memoria no volátil que almacena datos de forma permanente o semipermanente , como instrucciones de arranque y firmware que no se borran cuando se apaga el dispositivo .

15.DISCO DURO (HDD)

Dispositivos de almacenamiento de datos no volátil que usan platos magnéticos giratorios para guardar y recuperar información digital .

16.UNIDAD DE ESTADO SÓLIDO (SSD)

Dispositivo de almacenamiento de datos que usa memoria flash para guardar información de partes móviles .

17.CPU ( UNIDAD CENTRAL DE PROCESAMIENTO )

Es el cerebro de un dispositivo informático actuando como centro de control que ejecuta instrucciones realiza cálculos y procesa datos para el sistema operativo y las aplicaciones .

18.GPU ( UNIDAD DE PROCESAMIENTO GRÁFICO )

Circuito electrónico especializado en la informática diseñado para acelerar la creación de imágenes y videos

19.ARQUITECTURA VON NEUMANN

Diseño informático propuesto por John Von Neumann que usa una memoria única para almacenar tanto los datos como las instrucciones del programa y un bus compartido para acceder a ambos .

20.ARQUITECTURA HARVARD

Modelo informático que usa memorias y buses físicamente separados para el almacenamiento de instrucciones y datos .

21.BIT

Es la unidad más pequeña de información digital y su nombre es el acrónimo de binary digit (dígito binario ) representando un solo valor de los estados posibles 0 y 1 o un interruptor de encendido o apagado , verdadero o falso , o carga eléctrica .

22.BYTE

Unidad de información compuesta por ocho bits que son la unidad más pequeña de datos (0 o 1) los byte puede almacenar un carácter un número o una instrucción y es el pilar para que las computadoras almacenen y procesen información .

23.PALABRA (WORD )

Se refiere principalmente a microsoft word , un procesador de textos para crear, editar, y dar formato a el documento , pero también puede designar una unidad básica de datos en la arquitectura de un ordenador cuya longitud varía según la máquina .

24.DIRECCIÓN DE MEMORIA

Identificador único que señala una ubicación específica dentro de la memoria de la computadora .

25.BUS DE DATO

Vía de comunicación electrónica en una computadora o sistema compuesta por conductores que transportan datos en forma de bits (1 Y 0 )entre los diferentes componentes como la CPU, la memoria y los dispositivos periféricos .>

26.BUS DE CONTROL

Vía electrónica que transporta señales de control para coordinar y sincronizar las operaciones de los diferentes componentes de un sistema informático como la CPU , la memoria o los dispositivos de entrada/salida .

27.BUS DE DIRECCIONES

Conjunto de líneas eléctricas que transportan información de direcciones de memoria desde la CPU hacia otros componentes del sistema como la memoria RAM

28.REGISTRO (REGISTER )

Puede ser una estructura de datos que agrupa campos de distintos tipos , una memoria de alta velocidad dentro de el procesador (CPU) que almacena datos temporalmente e instrucciones para un procesamiento rápido, un conjunto de datos relacionados que forman una fila en una tabla de base de datos, o un archivo de eventos (log) que documenta las actividades de un sistema.

29.PILA (STACK)

Estructura de datos lineal que sigue el principio LIFO (por sus siglas en inglés, "Last In, First Out" o "último en entrar, primero en salir"). Esto significa que el último elemento que se añade a la pila es el primero que se puede extraer de ella, similar a una pila de platos o libros.

30.ESTUCTURA DE DATOS

Estructura de datos lineal que opera bajo el principio de primero en entrar, primero en salir FIFO - First-In, First-Out. Los elementos se añaden por un extremo llamado trasera o rear y se eliminan por el extremo opuesto el frente o front.

31.LISTA ENLAZADA

Forma específica de organizar, almacenar y manipular datos en un sistema para que puedan ser utilizados eficientemente. Permite gestionar y acceder a la información de manera organizada, lo que facilita la optimización del código y mejora el rendimiento de las aplicaciones, siendo fundamental para el diseño de software eficiente.

32.ARBOL BINARIO

Estructura de datos jerárquica donde cada nodo tiene un máximo de dos hijos: uno izquierdo y uno derecho. Esta limitación hace que los algoritmos de búsqueda, inserción y eliminación sean eficientes y permite organizar datos de forma ordenada

33.GRAFO

Estructura de datos abstracta compuesta por nodos (o vértices) y aristas (o arcos) que los conectan, sirviendo para modelar relaciones complejas entre objetos, como redes sociales, mapas de rutas o dependencias entre tareas.

34.TABLA HASH

Estructura de datos que almacena pares de clave-valor y utiliza una función hash para calcular una dirección o índice donde se guardarán los datos. Su principal ventaja es la eficiencia para buscar, insertar y eliminar datos, ya que puede encontrar un elemento en un tiempo constante, a diferencia de otras estructuras como las listas o los árboles.

35.ALGORITMO DE ORDENAMIENTO (SORTING )

Conjunto de instrucciones que toma una lista o arreglo de elementos como entrada y los organiza en un orden específico, como numérico o alfabético (lexicográfico), ya sea en orden ascendente o descendente.

36.ALGORITMO DE BÚSQUEDA

Método o conjunto de instrucciones para encontrar eficientemente un elemento específico (la "solución") dentro de un gran conjunto de datos o un espacio de posibles soluciones.

37.COMPLEJIDAD ALGORÍTMICA

Mide la eficiencia de un algoritmo, cuantificando los recursos (tiempo y espacio) que consume en función del tamaño de la entrada del problema.

38.BIG O NOTATION

Describe la eficiencia de un algoritmo midiendo cómo su tiempo de ejecución o uso de memoria escalan a medida que aumenta el tamaño de los datos de entrada (n).

39.RECURSIVIDAD

Técnica de programación donde una función se llama a sí misma para resolver un problema, desglosándolo en subproblemas más pequeños y similares hasta llegar a una condición de parada simple o caso base.

40.PROGRAMACIÓN ORIENTADA A OBJETOS (POO)

Es un paradigma de programación que organiza el código alrededor de objetos, los cuales son entidades que combinan datos (atributos) y comportamiento (métodos), en lugar de enfocarse únicamente en lógica y funciones.

41.HERENCIA

Es un pilar de la programación orientada a objetos (POO) que permite a una clase (llamada clase hija o derivada) crear nuevas instancias a partir de otra clase (clase padre o superclase) heredando sus características (atributos) y comportamientos (métodos).

42.ABSTRACCION

Proceso de ocultar los detalles complejos de un sistema para mostrar solo la información esencial, permitiendo a un usuario o programa interactuar con él de manera más simple y manejable.

43.POLIMORFISMO

Proceso de ocultar los detalles complejos de un sistema para mostrar solo la información esencial, permitiendo a un usuario o programa interactuar con él de manera más simple y manejable.

44. PROGRAMACIÓN FUNCIONAL

Es un estilo o paradigma de programación que trata la computación como la evaluación de funciones matemáticas, evitando los cambios de estado y los datos mutables.

45.PROGRAMACIÓN IMPERATIVA

Es un paradigma de programación que consiste en describir explícitamente a la computadora los pasos exactos que debe seguir para alcanzar un resultado deseado, actuando como una receta detallada para el programa.

46.PROGRAMACIÓN DECLARATIVA

Es un estilo de programación que se enfoca en describir qué quieres lograr, en lugar de cómo hacerlo, es decir, se especifica el resultado deseado y el sistema determina los pasos para alcanzarlo.

47.MÁQUINA VIRTUAL

Computadora definida por software que emula las funciones de una computadora física, ejecutando su propio sistema operativo y aplicaciones con recursos virtualizados.

48.EMULADOR

Aplicación de software (o, menos comúnmente, hardware) que simula el funcionamiento de otro dispositivo o sistema, permitiendo que software diseñado para uno funcione en una plataforma diferente.

49.PLACA BASE (MOTHERBOARD)

Placa de circuito impreso principal de una computadora, que actúa como el centro de conexión para todos los componentes internos y periféricos, permitiéndoles comunicarse entre sí y recibir energía para funcionar.

50.CHIPSET

Conjunto de chips (circuitos integrados) en la placa base de una computadora que actúa como controlador de tráfico para gestionar la comunicación entre la CPU (procesador), la memoria RAM, la tarjeta gráfica y otros dispositivos periféricos, como discos duros, puertos USB, y audio.

51.FUENTE DE ALIMENTACIÓN

Es el componente que convierte la corriente alterna (CA) de la red eléctrica en corriente continua (CC) de bajo voltaje que los demás componentes internos de un ordenador (como la CPU, la tarjeta gráfica y los discos duros) necesitan para funcionar.

52.SOCKET DE CPU

Es el componente físico en la placa base donde se instala el microprocesador (CPU). Su función principal es proporcionar la conexión mecánica y eléctrica entre la CPU y el resto del sistema, permitiendo la comunicación sin necesidad de soldar la CPU y facilitando su reemplazo.

53.DISIPADOR TÉRMICO

Componente de hardware que extrae y disipa el calor de los componentes electrónicos de alta potencia, como el procesador (CPU) o la tarjeta gráfica, para evitar el sobrecalentamiento y mantener un rendimiento óptimo y una mayor longevidad del equipo.

54.VENTILADOR

Es un componente que genera flujo de aire para la refrigeración activa de los componentes internos de una computadora, como la CPU y la GPU.

55.REFRIGERACION LIQUIDA

Técnica informática que utiliza un líquido, normalmente agua, para absorber y disipar el calor de componentes de alto rendimiento como el procesador (CPU) y la tarjeta gráfica (GPU).

56.GPU DEDICADA

Es un chip de procesador gráfico independiente de la CPU, con su propia memoria de video (VRAM) y sistema de refrigeración.

57.GPU INTEGRADA

Es un chip gráfico que está incorporado dentro del mismo procesador (CPU) o en la placa base, compartiendo la memoria RAM del sistema con el CPU en lugar de tener memoria dedicada propia

58.UNIDAD ÓPTICA

Componente de hardware informático que usa luz láser para leer y escribir datos en discos ópticos como CD, DVD y Blu-ray. Permite a los usuarios reproducir música y películas, instalar software desde discos y crear sus propias copias de seguridad de datos en discos vírgenes. Estas unidades utilizan un láser para crear y leer patrones de diminutos hoyos o hendiduras en la superficie del disco, los cuales representan datos binarios.

59.TARJETA DE SONIDO

Componente informático que permite la entrada y salida de audio en una computadora, convirtiendo señales analógicas (como las de un micrófono o altavoz) en señales digitales para el procesamiento de la computadora, y viceversa, para la reproducción.

60.TARJETA DE RED

Componente de hardware que permite a una computadora conectarse a una red, ya sea una LAN (red de área local) o Internet.

61.ADAPTADOR WI-FI

Es un componente de hardware que permite a un dispositivo, como una computadora o una consola, conectarse a una red inalámbrica (Wi-Fi) sin necesidad de cables.

62.BLUETOOTH

Tecnología de comunicación inalámbrica de corto alcance que permite a dispositivos como computadoras, teléfonos y periféricos conectarse y transferir datos entre sí sin cables.

63.PUERTO USB

Es una interfaz de conexión universal que permite conectar dispositivos electrónicos a ordenadores o a otros aparatos, posibilitando tanto la transferencia de datos como la carga de energía.

64.NANOTECNOLOGIA APLICADA EN TI

uso de la manipulación de la materia a escala nanométrica para mejorar la informática y la electrónica

65.THUNDERBOLT

interfaz de hardware de alta velocidad que combina la transferencia de datos, la salida de video y el suministro de energía en un único puerto

66.HDMI

Es un conector digital que transmite simultáneamente señales de audio y video de alta calidad entre dispositivos como ordenadores, televisores, consolas y proyectores.

67.DISPLAYPORT

Interfaz digital estándar desarrollada por VESA para conectar computadoras y otros dispositivos compatibles a pantallas como monitores y proyectores .

68.VGA

Es un estándar de interfaz analógica para la salida de video de un ordenador, que incluye un conector de 15 pines, una resolución de pantalla estándar (originalmente 640x480 píxeles) y el tipo de señal analógica usada para conectar el ordenador a monitores, proyectores y televisores.

69.DVI

Es un tipo de conector de vídeo que transmite señales de imagen digital de una computadora a un monitor.

70.RANURA PCIE

Conector físico en la placa base de una computadora que permite conectar componentes de alta velocidad, como tarjetas gráficas, unidades SSD, tarjetas de red y otras tarjetas de expansión, para mejorar el rendimiento y la funcionalidad del sistema.

/>/>

71.RANURA M.2

Interfaz compacta en la placa base de una computadora para conectar dispositivos de alta velocidad como unidades SSD, módulos Wi-Fi o Bluetooth.

72.BATERIA CMOS

Es una pila de botón, generalmente CR2032, que se encuentra en la placa base de un ordenador y suministra energía a la memoria CMOS y al reloj en tiempo real (RTC) para que guarden la configuración de hardware, la fecha y la hora incluso cuando el equipo está apagado o desconectado de la corriente.

73.MEMORIA ECC

Es un tipo de memoria RAM que detecta y corrige automáticamente errores de un solo bit, protegiendo así la integridad de los datos en sistemas críticos como servidores y estaciones de trabajo.

74.MEMORIA NO VOLÁTIL

Es un tipo de almacenamiento de datos que retiene la información incluso cuando la fuente de energía se interrumpe.

75.RAID 0

Es un nivel de tecnología RAID que combina dos o más discos duros para crear una unidad lógica única, dividiendo los datos en fragmentos (stripes) y distribuyéndolos secuencialmente entre los discos.

76.RAID 1

Es una configuración de almacenamiento de datos que crea una copia exacta de un conjunto de datos en dos o más discos.

77.RAID 5

Tecnología que usa entre tres y dieciocho discos duros para combinar el fraccionamiento de datos y la paridad distribuida, logrando un buen equilibrio entre el rendimiento de lectura y la tolerancia a fallos, permitiendo la recuperación de un disco al ser reemplazado, pero no tolera la falla de múltiples discos simultáneamente.

78.RAID 10

Es una tecnología que combina RAID 0 (striping o fragmentación) y RAID 1 (mirroring o duplicación), ofreciendo lo mejor de ambos mundos: alto rendimiento y fuerte tolerancia a fallos al duplicar primero los datos y luego distribuirlos en múltiples pares de discos.

79.NAS

Network Attached Storage (Almacenamiento Conectado a la Red). Es un dispositivo de alta capacidad que se conecta a una red, permitiendo que varios usuarios y dispositivos autorizados almacenen y recuperen datos de forma centralizada, usando protocolos como NFS y CIFS a través de una conexión Ethernet o Wi-Fi.

80.SAN

Red de Área de Almacenamiento (Storage Area Network, por sus siglas en inglés), que es una red de alta velocidad dedicada a conectar servidores con dispositivos de almacenamiento para centralizar los datos. Las SAN almacenan información en bloques y permiten que múltiples servidores accedan a un conjunto de almacenamiento compartido, lo que aumenta la eficiencia, la escalabilidad y la redundancia de los datos en un entorno empresarial.

81.SERVIDOR BLADE

Módulo informático delgado y modular que se inserta en un chasis compartido que proporciona alimentación, refrigeración y red a múltiples servidores individuales de alta densidad.

82.RACK

Marco o gabinete metálico estandarizado que se utiliza para alojar y organizar de forma compacta y segura equipos informáticos y de telecomunicaciones, como servidores, routers y switches.

83.CABLE DE PAR TRENZADO

Cable de red compuesto por pares de hilos de cobre aislados que se trenzan entre sí para reducir la interferencia electromagnética (EMI) y la diafonía.

84.CABLE COAXIAL

Cable eléctrico de alta frecuencia con un conductor central (generalmente de cobre) rodeado por una capa aislante, una malla metálica de blindaje y una cubierta exterior plástica. Su diseño proporciona una excelente resistencia a la interferencia y una transmisión de señales eficiente, lo que lo hizo muy popular en redes locales (LAN) y para la transmisión de televisión, aunque su uso está disminuyendo en favor de otras tecnologías.

85.FIBRA ÓPTICA

Tecnología de transmisión de datos que utiliza pulsos de luz para enviar información a través de hilos de vidrio o plástico extremadamente delgados. Su funcionamiento se basa en el principio de la reflexión interna total, donde la luz rebota dentro del núcleo de la fibra, permitiendo que los datos viajen a altas velocidades y grandes distancias con mínima pérdida de señal.

86.ANTENA

Componente diseñado para transmitir y recibir ondas electromagnéticas, como las señales Wi-Fi o de telefonía móvil, permitiendo la comunicación inalámbrica entre dispositivos.

87.ROUTER

Dispositivo que conecta redes distintas, como tu red doméstica (LAN) con Internet (WAN), y dirige el tráfico de datos para asegurar que la información llegue a su destino correcto.

88.SWITCH

Dispositivo de hardware que conecta múltiples equipos en una red local (LAN), permitiendo la comunicación eficiente y directa entre ellos.

89.HUB

dispositivo de red que conecta varios ordenadores o dispositivos para que compartan datos.

90.ACCESS POINT

Dispositivo que permite a los dispositivos con capacidad inalámbrica (como teléfonos, laptops, etc.) conectarse a una red cableada existente o a Internet, actuando como una puerta de enlace virtual y creando una red inalámbrica para compartir.

91.FIREWALL FISICO

Es un dispositivo de seguridad de red autónomo que funciona como una barrera para proteger una red. Se ubica en el límite de una red privada y actúa como un guardia vigilante, inspeccionando y filtrando el tráfico de datos entrante y saliente basándose en un conjunto preestablecido de reglas de seguridad, para bloquear accesos no autorizados y amenazas cibernéticas.

92.SERVIDOR BARE-METAL

Es una máquina física dedicada exclusivamente a un único cliente, sin compartir recursos, lo que permite un control total sobre la infraestructura y el sistema operativo para aplicaciones de alto rendimiento o con requisitos de seguridad específicos

93.CLúSTER DE SERVIDORES

Grupo de dos o más servidores interconectados que trabajan conjuntamente como un único sistema para ofrecer mayor disponibilidad, escalabilidad y rendimiento de las aplicaciones y servicios.

94.SUPERCOMPUTADORA

Sistema informático excepcionalmente potente y de alto rendimiento, diseñado para procesar enormes cantidades de datos y resolver cálculos complejos a velocidades muy altas, a menudo trabajando en paralelo con miles de procesadores

95.MAINFRAME

Es un sistema de computación de alto rendimiento, seguridad y confiabilidad diseñado para procesar grandes volúmenes de datos y realizar millones de transacciones en tiempo real, siendo fundamental para organizaciones en sectores como banca, seguros, gobierno y logística que requieren una operación continua y segura de sus servicios críticos.

96.IOT DEVICE

Objeto físico equipado con sensores, software y otras tecnologías que le permiten conectarse a Internet para recopilar e intercambiar datos con otros dispositivos y sistemas.

97.MICROCONTROLADOR

Pequeña computadora de un solo chip que integra en un único circuito integrado el procesador (CPU), memoria (RAM y ROM) y periféricos de entrada/salida (E/S), diseñado para tareas específicas en sistemas integrados, controlando procesos y automatizando funciones en dispositivos electrónicos de uso diario e industrial.

98.FPGA

(Field-Programmable Gate Array) o matriz de puertas programables en campo es un circuito integrado que permite a los usuarios configurar su arquitectura de hardware después de la fabricación para realizar una amplia variedad de tareas.

99.LAN

Es probablemente un error tipográfico de LAN en informática, que se refiere a una Red de Área Local (del inglés Local Area Network). Es una red que conecta un número limitado de dispositivos (como ordenadores, impresoras o teléfonos) en un área geográfica pequeña y específica, como una oficina, hogar o edificio, para que puedan compartir recursos y comunicarse entre sí.

100.WAN

Significa Red de Área Extensa (Wide Area Network) y es una red informática que conecta múltiples redes locales (LAN) que abarcan un área geográfica grande, como ciudades o países.

101.MAN

Red de Área Metropolitana (Metropolitan Área Network), que es una red de datos de alta velocidad que interconecta varias redes de área local (LAN) dentro de un área geográfica extensa como una ciudad o un pueblo, siendo mayor que una LAN y menor que una red de área amplia (WAN).

102.PAN

Significa Personal Area Network (Red de Área Personal) y se refiere a una red que conecta dispositivos electrónicos personales dentro del espacio de trabajo o alcance de un usuario individual, generalmente de unos pocos metros.

103.VPN

(Red Privada Virtual) es una tecnología que crea una conexión segura y cifrada a través de Internet para tu dispositivo, actuando como un "túnel" digital entre tu equipo y un servidor VPN remoto. Esto enmascara tu dirección IP real, cifra tus datos para proteger tu privacidad y te permite acceder a redes privadas de manera segura, mejorando la seguridad y el anonimato en línea.

104.VLAN

(Red de Área Local Virtual) es una tecnología que permite dividir un único switch de red física en múltiples redes lógicas o virtuales independientes. Esto crea segmentos de red aislados que mejoran la seguridad y el rendimiento, ya que el tráfico de cada VLAN solo puede comunicarse dentro de su propio segmento o con otras VLAN a través de un enrutador.

105.TOPOLOGÍA EN ESTRELLA

Estructura de red donde todos los dispositivos (nodos) están conectados a un dispositivo central, como un concentrador (hub) o un conmutador (switch).

106.TOPOLOGÍA EN BUS

diseño de red en el que todos los dispositivos se conectan a un único cable principal, llamado "bus"

107.OPOLOGÍA EN ANILLO

Configuración de red en la que los dispositivos se conectan en un bucle cerrado, formando un camino circular para la transmisión de datos. Los datos viajan por el anillo pasando de un dispositivo a otro hasta llegar a su destino, cada dispositivo está conectado a otros dos. Los anillos pueden ser unidireccionales o bidireccionales.

108.TOPOLOGÍA EN MALLA

Configuración de red donde los dispositivos están interconectados de forma descentralizada, con cada nodo conectado a varios otros para crear múltiples rutas para la transmisión de datos.

109.MODELO OSI

(Interconexión de Sistemas Abiertos) es un marco conceptual estándar de siete capas que divide las funciones de comunicación de una red informática para permitir que distintos sistemas se comuniquen entre sí de manera eficiente.

110.CAPA FÍSICA

Componente de hardware de una red informática que se encarga de la transmisión y recepción de bits de datos sin procesar a través de un medio físico, como cables de cobre, fibra óptica o el aire. Define los estándares de hardware, como conectores, voltajes y frecuencias, y los componentes activos como tarjetas de red y repetidores, asegurando que los datos puedan moverse de un punto a otro en forma de señales eléctricas u ópticas.

111.CAPA DE ENLACE DE DATOS

Es la segunda capa del modelo OSI, responsable de la transferencia de datos entre dos dispositivos directamente conectados en la misma red local (LAN).

112.CAPA DE RED

Capa conceptual en los modelos de redes informáticas, como el Modelo OSI, que se encarga del enrutamiento, reenvío y direccionamiento de datos para que puedan viajar a través de distintas redes y llegar a su destino.

113.CAPA DE TRANSPORTE

Es la cuarta capa del modelo OSI y es responsable de la comunicación de extremo a extremo entre aplicaciones en diferentes dispositivos (hosts), utilizando puertos para dirigir los datos a la aplicación correcta.

114.CAPA DE SESIÓN

Es la quinta capa del modelo OSI en informática y es la responsable de establecer, mantener y finalizar conexiones (sesiones) entre aplicaciones que se comunican en una red, garantizando una comunicación organizada y sincronizada entre los sistemas.

115.CAPA DE PRESENTACIÓN

Es la sexta capa del modelo OSI encargada de preparar los datos para la capa de aplicación, traduciendo los diferentes formatos de datos y codificaciones entre sistemas para asegurar que sean entendidos.

116.CAPA DE APLICACIÓN

Es la interfaz directa entre las aplicaciones de software que usa el usuario y los servicios de red subyacentes, siendo la capa más alta (la séptima) del modelo OSI.

117.MODELO TCP/IP

Conjunto de protocolos de comunicación que define cómo los dispositivos envían y reciben datos a través de redes y es el fundamento de Internet, permitiendo el intercambio de información de manera fiable y eficiente.

118.PROTOCOLO IP

Conjunto de reglas que define cómo los datos se envían y reciben entre dispositivos a través de las redes, incluyendo Internet.

119.IPV4

H2>

Formato de dirección estándar para la comunicación en redes e Internet, que asigna a cada dispositivo una dirección numérica única de 32 bits, escrita como cuatro grupos de números separados por puntos.

120.IPV6

Es la versión más reciente del protocolo de Internet que proporciona direccionamiento único para miles de millones de dispositivos, a diferencia de su predecesor IPv4, debido a su gran espacio de direcciones (128 bits).

121.TCP

Es un estándar de comunicación que descompone los datos en paquetes, los envía a través de una red, y se asegura de que lleguen al destino en el orden correcto y sin errores, gracias a su naturaleza de conexión fiable y orientada.

122.UDP

Es un protocolo de transporte de Internet que se usa para la comunicación rápida y en tiempo real, priorizando la velocidad sobre la fiabilidad porque no establece una conexión formal ni garantiza la entrega o el orden de los datos.

.ICMP

Es un protocolo de la capa de red que se utiliza para enviar mensajes de error y control, diagnosticar problemas de comunicación en redes y verificar la conectividad entre dispositivos.

123.ARP

Es un protocolo de red fundamental que traduce una dirección IP (lógica, que puede cambiar) a una dirección MAC (física, que es única y fija), permitiendo que los dispositivos se comuniquen en una red local.

124.NAT

Es la sigla de Network Address Translation (Traducción de Direcciones de Red), una tecnología informática que permite que varios dispositivos en una red privada (con Direcciones IP internas) compartan una única dirección IP pública para acceder a Internet. Esta función, implementada en un enrutador, actúa como intermediaria, traduciendo las direcciones y puertos de los paquetes de datos y preservando así el escaso espacio de direcciones IPv4, además de ofrecer una capa de seguridad al ocultar las direcciones IP internas.

125.DHCP

(Protocolo de Configuración Dinámica de Host) es un protocolo de red que asigna automáticamente direcciones IP y otros parámetros de configuración (como la máscara de subred, puerta de enlace y direcciones DNS) a los dispositivos (clientes) que se conectan a una red.

126.DNS

(Sistema de Nombres de Dominio) es el servicio que traduce nombres de dominio fáciles de recordar para los humanos (como www.ejemplo.com) en direcciones IP numéricas que las computadoras usan para identificar y conectar dispositivos en una red, actuando como una guía telefónica de Internet. Este proceso es fundamental para navegar por la web, ya que permite a los navegadores encontrar y cargar los sitios web correctos.

127.HTTP

(Protocolo de Transferencia de Hipertexto) es un protocolo de comunicación que define cómo los navegadores web (clientes) envían solicitudes a los servidores web para obtener y mostrar contenido, y cómo los servidores responden a esas solicitudes, posibilitando el funcionamiento de la World Wide Web.

128.HTTPS

(Protocolo de Transferencia de Hipertexto Seguro) es una extensión del protocolo HTTP que añade una capa de seguridad para cifrar la comunicación entre el navegador de un usuario y un servidor web, utilizando protocolos como SSL/TLS.

129.FTP

(Protocolo de Transferencia de Archivos) es un estándar de red que permite transferir archivos entre dos computadoras en una red, como Internet. Funciona mediante un modelo cliente-servidor, donde el cliente inicia la conexión y solicita archivos, y el servidor los proporciona, permitiendo tanto la carga (subir) como la descarga (bajar) de información.

130.SFTP

Es el acrónimo de SSH File Transfer Protocol o Protocolo de Transferencia Segura de Archivos. Se trata de un protocolo que utiliza SSH (Secure Shell) para transferir archivos de manera segura, cifrando los datos durante su transmisión para protegerlos de amenazas y garantizar una comunicación segura. A diferencia de FTP, SFTP requiere clientes y servidores SFTP específicos para su funcionamiento y se utiliza para el intercambio seguro de información entre sistemas.

131.SSH

Secure Shell es un protocolo de red que permite la comunicación segura y el acceso remoto a sistemas. Establece conexiones cifradas entre ordenadores, lo que protege las contraseñas y los datos de ser interceptados en redes inseguras. Se utiliza principalmente para administrar sistemas remotos, transferir archivos de forma segura y ejecutar comandos, ofreciendo una alternativa segura a protocolos menos seguros como Telnet o FTP.

132.TELNET

Es un protocolo de red antiguo, basado en TCP/IP, que permite el acceso y control remoto de un ordenador a través de una interfaz de línea de comandos, como si el usuario estuviera presente frente al equipo.

133.SMTP

(Del inglés, Simple Mail Transfer Protocol o Protocolo Simple de Transferencia de Correo) es un protocolo estándar que se utiliza para el envío y la recepción de correos electrónicos a través de Internet.

134.POP3

(Post Office Protocol versión 3) es un protocolo que permite descargar correos electrónicos de un servidor a un dispositivo local, como una computadora o teléfono, para su visualización y almacenamiento, y que, por defecto, los elimina del servidor una vez descargados.

135.IMAP

Protocolo de Acceso a Mensajes de Internet, un protocolo para recibir correos electrónicos que permite acceder a tus mensajes desde múltiples dispositivos y los almacena en el servidor remoto.

136.SNMP

(Protocolo Simple de Administración de Red) es un protocolo de capa de aplicación de la familia TCP/IP que permite monitorear y gestionar dispositivos de red como routers, switches y servidores.

137.BGP

Protocolo de Puerta de Enlace Fronteriza (Border Gateway Protocol), es el protocolo de enrutamiento fundamental de Internet que permite que las redes intercambien información sobre cómo llegar a otros destinos, funcionando como el "servicio postal" de la web para encontrar las rutas más eficientes para el tráfico de datos.

138.OSPF

(Open Shortest Path First) es un protocolo de enrutamiento de estado de enlace que utiliza el algoritmo de Dijkstra (SPF) para calcular y determinar la ruta más eficiente para el tráfico de datos dentro de un sistema autónomo (AS).

139. PROTOCOLO RIP

(Routing Information Protocol, Protocolo de Información de Enrutamiento) es un protocolo de enrutamiento dinámico de vector de distancia que los enrutadores utilizan para descubrir y compartir información sobre rutas en una red, usando el recuento de saltos como métrica principal para elegir la ruta más corta, y tiene un límite máximo de 15 saltos.

140.MPLS

(Conmutación de etiquetas multiprotocolo) es una tecnología de red que optimiza el enrutamiento de datos asignando etiquetas a los paquetes en lugar de usar direcciones IP.

141.QOS

(Calidad de Servicio, del inglés Quality of Service) es un conjunto de técnicas y tecnologías que permiten priorizar ciertos tipos de tráfico en una red para garantizar el rendimiento de aplicaciones críticas, como la voz o el video, sobre otras menos importantes, incluso cuando hay congestión o ancho de banda limitado.

142.LATENCIA

Es el retraso o el tiempo que tardan los datos en viajar entre dos puntos de una red o en ser procesados por un sistema, medido comúnmente en milisegundos (ms).

143.JITTER

Variación o fluctuación irregular en el tiempo de llegada de los paquetes de datos a través de una red de Internet.

144.ANCHO DE BANDA

Cantidad máxima de datos que pueden transmitirse a través de una conexión de red en un período de tiempo determinado, como por segundo.

145.PACKET LOSS

("Packet loss" en inglés) en informática se refiere a la situación en la que los paquetes de datos enviados a través de una red no logran llegar a su destino, lo que interrumpe la comunicación y afecta el rendimiento de la red. Esto puede causar retrasos, interrupciones en llamadas o transmisiones, y una experiencia de juego deficiente, ya que cada paquete es una pequeña unidad de datos necesaria para que la información se reciba correctamente.

146.TRACEROUTE

(O tracert en Windows) es una herramienta de diagnóstico de red que rastrea la ruta que siguen los paquetes de datos desde un origen hasta un destino, identificando cada router intermedio y midiendo la latencia en cada salto.

147.PING

Tiempo de respuesta de la comunicación entre dos dispositivos en una red (medido en milisegundos) como al comando que envía paquetes de datos para comprobar la conectividad y medir esa latencia.

148.SEGURIDAD PERIMETRAL

Definición correspondiente al concepto.

149.SEGURIDAD LÓGICA

Conjunto de medidas y controles (principalmente de software) que protegen los datos, los programas y los sistemas de accesos no autorizados, modificación y daños.

150.CRIPTOGRAFÍA

Conjunto de técnicas y algoritmos matemáticos que se utilizan para proteger la información, transformándola en un código ilegible para personas no autorizadas (cifrado) y asegurando que pueda ser entendida únicamente por el destinatario previsto (descifrado).

151.CRIPTOGRAFÍA SIMÉTRICA

Método de seguridad informática que utiliza la misma clave secreta para cifrar (codificar) y descifrar (decodificar) un mensaje o datos. Tanto el remitente como el receptor deben poseer la clave compartida para que el mensaje pueda ser leído. Es un proceso eficiente y rápido, ideal para grandes volúmenes de datos, pero su principal desafío es la distribución segura de esta clave secreta.

152.CRIPTOGRAFÍA ASIMÉTRICA

Es un sistema de seguridad informática que utiliza un par de claves matemáticamente relacionadas: una clave pública que se puede compartir libremente, y una clave privada que debe mantenerse secreta.

153.RSA

Algoritmo de criptografía de clave pública que se usa para asegurar la comunicación y las transacciones digitales, como el cifrado de mensajes y la creación de firmas digitales.

154.AES

Algoritmo de cifrado de bloques simétrico, considerado el estándar para proteger datos electrónicos sensibles. Utiliza la misma clave para cifrar y descifrar información, procesándola en bloques de 128 bits y permitiendo claves de 128, 192 o 256 bits para diferentes niveles de seguridad.

155.3DES

Algoritmo de cifrado simétrico que aplica el algoritmo DES tres veces, utilizando tres claves diferentes para aumentar la seguridad en comparación con el DES original.

156.SHA-256

Algoritmo de hash criptográfico que genera una huella digital de 256 bits (o 64 caracteres hexadecimales) a partir de cualquier dato de entrada, lo que lo hace único y unidireccional, por lo que no se puede revertir para obtener los datos originales.

157.HASH

Es el resultado de aplicar una función hash a un conjunto de datos, generando una cadena de caracteres de longitud fija y única que lo representa, utilizada para verificar la integridad y seguridad de la información.

158.SALTING

Técnica de seguridad que consiste en añadir una cadena de datos aleatoria y única (llamada "sal") a una contraseña antes de que esta sea sometida a un algoritmo de hash.

159.FIRMA DIGITAL

Sello electrónico y criptográfico que garantiza la autenticidad e integridad de un documento digital, confirmando que proviene del firmante y que no ha sido alterado.

160.CERTIFICADO DIGITAL

Es un archivo que funciona como una identidad electrónica para una persona, entidad o servidor, vinculando su clave pública a su información y siendo emitido por una Autoridad de Certificación de confianza.

161.PKI (INFRAESTRUCTURA DE CLAVE PÚBLICA)

Es un sistema de hardware, software, políticas y procedimientos que permite la emisión, gestión, distribución, uso y revocación de certificados digitales, asegurando la confidencialidad, integridad y autenticidad en las comunicaciones y transacciones electrónicas.

162.HTTPS/TLS

HTTPS es la combinación del protocolo HTTP y el protocolo de seguridad TLS (antes SSL), que se usa para asegurar la comunicación entre un navegador web y un sitio web.

163.SSL

Es un protocolo de seguridad, hoy en día obsoleto y reemplazado por su sucesor, TLS (Transport Layer Security), que permite el cifrado y la autenticación de las comunicaciones en Internet.

164.CIFRADO DE DISCO COMPLETO

Es el uso de un sistema de certificados digitales y claves criptográficas para proteger todo el contenido de un disco en un dispositivo, garantizando la seguridad y confidencialidad de los datos.

165.VPN CON IPSEC

Es una conexión segura y cifrada a través de una red pública como Internet, utilizando el conjunto de protocolos IPsec para proteger la comunicación.

166.TÚNEL SEGURO

Conexión cifrada que protege la transmisión de datos en una red, encapsulando un protocolo de red dentro de otro para ocultar el tráfico de la detección no autorizada.

167.FIREWALL DE RED

Es un dispositivo, ya sea físico (hardware) o un programa (software), que actúa como una barrera de seguridad entre una red privada y una red externa, como Internet, o entre diferentes segmentos de una misma red.

168.FIREWALL DE APLICACIÓN

Dispositivo de seguridad que protege las aplicaciones web de ataques comunes monitorizando y filtrando el tráfico HTTP/S en la capa 7 (capa de aplicación) de la red.

169.IDS (INTRUSION DETECTION SYSTEM)

Es una herramienta de seguridad informática que monitoriza el tráfico de red y los eventos del sistema en busca de actividades maliciosas o actividades que violen las políticas de seguridad.

170.IPS (INTRUSION PREVENTION SYSTEM)

Dispositivo o software de seguridad que monitorea el tráfico de red y toma acciones automáticas para bloquear o detener amenazas y accesos no autorizados en tiempo real, a diferencia de un IDS que sólo alerta sobre intrusiones.

171.SIEM

Es un sistema que recopila, agrega y analiza en tiempo real datos de seguridad de diversas fuentes (como aplicaciones, servidores, redes y usuarios) para detectar amenazas, investigar incidentes y simplificar el cumplimiento de normativas.

172.HONEYPOT

Un IDS (Sistema de Detección de Intrusiones) es un sistema de seguridad que monitorea y alerta sobre actividades sospechosas, mientras que un honeypot es un sistema trampa, un señuelo, diseñado para atraer y desviar a los ciberdelincuentes de los sistemas legítimos, permitiendo a los investigadores de seguridad recopilar información sobre sus métodos.

173.INGENIERÍA SOCIAL

Es una táctica de manipulación psicológica utilizada por ciberdelincuentes para engañar a las personas y obtener información confidencial o control sobre sus sistemas. Se aprovecha de la confianza, la curiosidad o el miedo de los usuarios para inducirlos a realizar acciones que comprometen su seguridad, como revelar contraseñas o descargar software malicioso.

174.PHISHING

Es un tipo de ciberataque donde los delincuentes se hacen pasar por una entidad de confianza (como un banco o una empresa) para engañar a las personas y obtener información personal y financiera sensible, como contraseñas o datos de tarjetas de crédito.

175.RANSOMWARE

En el ransomware de extorsión doble, los atacantes no solo cifran archivos, sino que también roban datos confidenciales y amenazan con liberarlos públicamente si no se paga el rescate. Con la destrucción de datos, se amenaza con destruir los datos de la víctima si esta no paga el rescate.

176.MALWARE

Cualquier tipo de programa o código dañino diseñado para infiltrarse en un sistema informático, sistema operativo o red sin el consentimiento del usuario, con el objetivo de interrumpir su funcionamiento, robar información sensible, obtener acceso no autorizado o causar daños.

177.SPYWARE

Es un tipo de software malicioso que se instala en un dispositivo (como una computadora o teléfono) sin el consentimiento del usuario para recopilar información privada y enviarla a terceros.

178.ADWARE

Es un tipo de malware o software potencialmente no deseado que muestra publicidad no autorizada en tu dispositivo.

179.TROYANO

Es un tipo de malware que se disfraza de software legítimo o útil para engañar al usuario e infiltrarse en su sistema. Una vez instalado, el troyano permite al atacante robar información, espiar al usuario, tomar el control remoto del dispositivo o instalar otro tipo de malware, como ransomware o spyware.

180.ROOTKIT

Es un tipo de malware que otorga a un atacante acceso y control privilegiado sobre un sistema informático sin ser detectado, ocultando su propia presencia mediante la alteración de procesos, archivos y registros del sistema operativo. Su capacidad para pasar desapercibido lo convierte en una de las amenazas más difíciles de detectar y eliminar, permitiendo a los ciberdelincuentes robar datos confidenciales, instalar otro malware o utilizar el equipo infectado para lanzar ataques.

181.BOTNET

Red de ordenadores, dispositivos de IoT y otras máquinas infectadas con software malicioso (malware) y controladas de forma remota por un ciberdelincuente conocido como "botmaster" o "bot-herder".

182.ATAQUE DDOS

Es un ciberataque que busca sobrecargar un servidor, sitio web o red con una gran cantidad de tráfico falso, proveniente de múltiples fuentes maliciosas, con el objetivo de agotar sus recursos y hacer que se vuelva inaccesible o no esté disponible para los usuarios legítimos.

183.ATAQUE MITM (MAN IN THE MIDDLE)

(Man in the Middle o de Hombre en el Medio) es un ciberataque en el que un atacante se interpone entre dos partes que se están comunicando para interceptar, robar, leer o modificar sus comunicaciones, a menudo sin que ninguna de las partes se dé cuenta. Este tipo de ataque busca robar información sensible como contraseñas, datos bancarios o información personal al espiar las comunicaciones y puede realizarse mediante redes Wi-Fi maliciosas, suplantación de identidad o certificados digitales falsos.

184.ATAQUE DE DICCIONARIO

Es una técnica para descifrar contraseñas que consiste en probar sistemáticamente una lista de palabras, frases y combinaciones comunes de un "diccionario" de contraseñas, buscando encontrar la correcta para acceder a un sistema protegido. Este método aprovecha la tendencia de los usuarios a elegir contraseñas predecibles, y se considera una forma de ataque de fuerza bruta.

185.ATAQUE DE INYECCIÓN SQL

Es un ciberataque que explota vulnerabilidades en las aplicaciones web al insertar comandos SQL maliciosos en los campos de entrada de datos, como nombres de usuario o contraseñas. Cuando estos comandos son ejecutados por la base de datos, un atacante puede obtener acceso no autorizado, manipular o eliminar información confidencial, o incluso comprometer el sistema completo.

186.CROSS-SITE SCRIPTING (XSS)

Es un tipo de ataque informático que consiste en inyectar código malicioso (generalmente JavaScript) en sitios web de confianza, para que luego sea ejecutado en el navegador de otros usuarios finales, lo cual puede llevar al robo de datos como cookies y sesiones, la suplantación de identidad o la redirección a sitios maliciosos. Los ataques XSS aprovechan las vulnerabilidades en las aplicaciones web para entregar el código dañino, que se ejecuta en el cliente sin que el usuario se dé cuenta.

187.CROSS-SITE REQUEST FORGERY (CSRF)

Es un ciberataque en el que un atacante engaña al navegador de un usuario autenticado para ejecutar acciones no deseadas en un sitio web en el que el usuario ya ha iniciado sesión. El atacante explota la confianza que el sitio web tiene en la autenticación del usuario (por ejemplo, a través de cookies de sesión) para enviar solicitudes maliciosas que el sitio web considera legítimas, llevando a la modificación de configuraciones, transferencia de fondos o realización de otras acciones autorizadas sin el consentimiento del usuario.

188.ZERO-DAY

Se refiere a una vulnerabilidad de seguridad desconocida para el creador del software y el público, lo que permite a los ciberatacantes explotarla antes de que se pueda lanzar una defensa.

189.ESCANEO DE PUERTOS

Técnica utilizada para identificar qué puertos de un dispositivo de red o servidor están abiertos, cerrados o filtrados, revelando qué servicios (como servidores web HTTP en el puerto 80 o servidores SSH en el puerto 22) se ejecutan y qué vulnerabilidades podrían existir.

190.PENTESTING

Es una práctica de ciberseguridad que simula ciberataques para identificar vulnerabilidades en sistemas, redes y aplicaciones de una organización.

191.ETHICAL HACKING

Las pruebas de penetración (también llamadas “pentesting”) son una práctica para poner a prueba un sistema informático, red o aplicación web para encontrar vulnerabilidades que un atacante podría explotar.

192.RED TEAM

Grupo de hackers éticos que simulan un ataque ciberdelincuente real contra una organización para probar y mejorar sus defensas de seguridad, su capacidad de detección y respuesta ante incidentes.

193.BLUE TEAM

Grupo de profesionales de ciberseguridad que se encarga de la defensa de los sistemas y redes de una organización, trabajando proactivamente para prevenir, detectar, contener y responder a incidentes de seguridad y ataques cibernéticos.

194.PURPLE TEAM

Es una estrategia que combina los esfuerzos de los equipos rojos (ofensivos) y equipos azules (defensivos) para mejorar la postura de seguridad de una organización.

195.SEGURIDAD EN LA NUBE

Conjunto de tecnologías, políticas y procesos que protegen los datos, aplicaciones e infraestructura de la nube contra amenazas de seguridad, accesos no autorizados y ataques cibernéticos.

196.SEGURIDAD EN IOT

Es la práctica de proteger dispositivos conectados a Internet (como cámaras, electrodomésticos inteligentes, etc.) y las redes a las que se conectan de amenazas y accesos no autorizados.

197.INTELIGENCIA ARTIFICIAL

Es un campo que crea sistemas capaces de realizar tareas que normalmente requieren inteligencia humana, como el aprendizaje, el razonamiento, la percepción y la resolución de problemas.

198.APRENDIZAJE AUTOMÁTICO (MACHINE LEARNING)

Es una rama de la inteligencia artificial que permite a los sistemas informáticos aprender y mejorar a partir de datos sin ser programados explícitamente. Utiliza algoritmos para identificar patrones en grandes conjuntos de datos, y luego usa esos patrones para hacer predicciones o tomar decisiones.

199.APRENDIZAJE SUPERVISADO

Método de aprendizaje automático donde un algoritmo se entrena con un conjunto de datos etiquetados, es decir, datos que ya tienen una salida correcta o conocida asociada. El objetivo es que el modelo aprenda la relación entre las entradas y salidas para poder predecir resultados correctos cuando se le presentan datos nuevos y nunca antes vistos.

200.APRENDIZAJE NO SUPERVISADO

Método de aprendizaje automático donde un algoritmo de inteligencia artificial analiza datos sin etiquetar para descubrir patrones y estructuras intrínsecas por sí mismo.

201.APRENDIZAJE POR REFUERZO

Es un área de la inteligencia artificial y el aprendizaje automático donde un agente, como un programa de computadora, aprende a tomar decisiones óptimas en un entorno al experimentar y recibir retroalimentación (recompensas o castigos) para lograr un objetivo.

202.DEEP LEARNING

Es una rama de la inteligencia artificial y el machine learning que utiliza redes neuronales artificiales con múltiples capas para identificar patrones complejos en grandes volúmenes de datos, especialmente no estructurados como imágenes, audio o texto.

203.RED NEURONAL ARTIFICIAL

Es un sistema computacional inspirado en el cerebro humano que aprende de los datos para identificar patrones complejos y resolver problemas de manera similar a como lo haría un biólogo, procesando información a través de múltiples capas de "neuronas" interconectadas.

204.RED CONVOLUCIONAL (CNN)

Es un tipo de red neuronal artificial diseñada para procesar datos con estructura de cuadrícula, como las imágenes, y se inspira en el sistema visual humano.

205.RED RECURRENTE (RNN)

Es un tipo de red neuronal artificial diseñada para procesar datos secuenciales, como texto o series temporales, utilizando una memoria interna para mantener información de pasos anteriores y predecir el siguiente paso en la secuencia.

206.LSTM

Es un tipo de red neuronal recurrente (RNN) que procesa datos secuenciales, permitiéndote recordar información importante durante largos períodos.

207.GAN (GENERATIVE ADVERSARIAL NETWORK)

Arquitectura de aprendizaje profundo que utiliza dos redes neuronales en competencia: un generador que crea datos nuevos y realistas, y un discriminador que intenta distinguir los datos falsos del generador de los datos reales de un conjunto de entrenamiento.

208.PERCEPTRÓN

Es el modelo computacional más simple de una red neuronal artificial. Se trata de un algoritmo de aprendizaje supervisado que funciona como un clasificador binario, utilizando un conjunto de entradas ponderadas para producir una única salida. Su principal función es aprender un «límite de decisión» lineal para clasificar datos, aunque su complejidad se puede ampliar con redes neuronales multicapa para tareas más complejas.

209.BACKPROPAGATION

Algoritmo de aprendizaje automático clave para entrenar redes neuronales artificiales, que calcula el error entre la salida predicha y la salida real, para luego propagar este error hacia atrás a través de la red.

210.GRADIENT DESCENT

Algoritmo de optimización que se utiliza comúnmente para entrenar modelos de aprendizaje automático y redes neuronales.

211.OVERFITTING

El sobreajuste es un comportamiento de machine learning no deseado que se produce cuando el modelo de machine learning brinda predicciones precisas para los datos de entrenamiento, pero no para los datos nuevos.

212.UNDERFITTING

Ocurre cuando un modelo de aprendizaje automático es demasiado simple para capturar los patrones complejos de los datos, lo que resulta en un mal rendimiento tanto en los datos de entrenamiento como en los datos no vistos.

213.CONJUNTO DE ENTRENAMIENTO

Conjunto de datos fundamental para el desarrollo de un modelo de aprendizaje automático. Es la parte de nuestro conjunto de datos que se utiliza para entrenar un algoritmo que identifica patrones, relaciones y estructuras dentro de los datos.

214.CONJUNTO DE VALIDACIÓN

Conjunto de datos que se utiliza para entrenar la inteligencia artificial (IA) con el objetivo de encontrar y optimizar el mejor modelo para resolver un problema determinado. Los conjuntos de validación también se conocen como conjuntos de desarrollo.

215.CONJUNTO DE PRUEBA

Un conjunto de pruebas (o test suite) es una colección organizada de casos de prueba individuales, scripts o incluso otros conjuntos de pruebas, diseñada para evaluar de manera específica y coordinada la funcionalidad, el rendimiento, la integración o la seguridad de un sistema o aplicación de software.

216.DATASET

Colección de datos organizados de forma estructurada para facilitar su análisis, procesamiento y manipulación.

217.FEATURE ENGINEERING

La ingeniería de características implica la extracción y transformación de variables a partir de datos sin procesar, como las listas de precios, las descripciones de los productos y los volúmenes de ventas, de modo que sea posible utilizar las características para el entrenamiento y la predicción.

218.SELECCIÓN DE CARACTERÍSTICAS

Proceso de seleccionar las características más relevantes de un conjunto de datos para usarlas al crear y entrenar un modelo de machine learning (ML).

219.REDUCCIÓN DE DIMENSIONALIDAD

Técnica de informática, especialmente en aprendizaje automático, que simplifica conjuntos de datos de alta dimensión transformándolos en un espacio de menor dimensión.

220.PCA (PRINCIPAL COMPONENT ANALYSIS)

Técnica de reducción de dimensionalidad que transforma un conjunto grande de variables en un conjunto más pequeño de variables nuevas, llamadas componentes principales, que retienen la mayor parte de la información original.

221.KNN (K-NEAREST NEIGHBORS)

Algoritmo de aprendizaje automático que se utiliza para clasificación y regresión.

222.ÁRBOL DE DECISIÓN

Algoritmo de aprendizaje automático supervisado, no paramétrico, que se utiliza tanto para clasificación como para regresión.

223.RANDOM FOREST

Algoritmo de aprendizaje automático supervisado que combina múltiples árboles de decisión para mejorar la precisión y la robustez de las predicciones en tareas de clasificación y regresión.

224.SVM (SUPPORT VECTOR MACHINE)

Algoritmo de aprendizaje automático supervisado que clasifica datos encontrando el hiperplano que mejor separa las diferentes clases, maximizando la distancia (margen) entre ellas.

225.NAIVE BAYES

Algoritmo de machine learning supervisado que clasifica datos basándose en el teorema de Bayes y la suposición de independencia condicional de las características.

226.REGRESIÓN LINEAL

Técnica estadística que predice el valor de una variable dependiente basándose en una o más variables independientes, ajustando una línea recta a los datos para encontrar una relación lineal.

227.REGRESIÓN LOGÍSTICA

Modelo estadístico de aprendizaje automático utilizado para predecir la probabilidad de un resultado binario (sí/no, 0/1) basándose en un conjunto de variables de entrada.

228.CLUSTERING

El clustering es un algoritmo de machine learning no supervisado que organiza y clasifica diferentes objetos, puntos de datos u observaciones en grupos.

229.K-MEANS

Algoritmo de aprendizaje automático no supervisado que agrupa un conjunto de datos en K clústeres o grupos. Su objetivo es minimizar la distancia entre cada punto de datos y el centroide (punto medio) de su clúster asignado, logrando que los puntos dentro de un mismo grupo sean lo más similares posible. El proceso es iterativo: se asignan puntos a centroides, se actualizan los centroides y se repite hasta que los clústeres se estabilizan.

233.DBSCAN

Algoritmo de agrupación (clustering) basado en la densidad que agrupa puntos de datos cercanos, identifica ruido (puntos aislados) y puede detectar clústeres de formas arbitrarias, a diferencia de algoritmos como K-Means, que solo encuentran clústeres con formas globulares.

230.ALGORITMOS EVOLUTIVOS

Son una clase de algoritmos de optimización y búsqueda basados en la inteligencia artificial que imitan los procesos de la evolución biológica.

231.ALGORITMOS GENÉTICOS

Método de optimización y búsqueda inspirado en la selección natural, que evoluciona una población de posibles soluciones (cromosomas) para encontrar las mejores soluciones a un problema.

232.PROCESAMIENTO DE LENGUAJE NATURAL (NLP)

Es un campo de la inteligencia artificial y la informática que permite a las computadoras entender, interpretar y generar lenguaje humano escrito y hablado.

233.TOKENIZACIÓN

Proceso que consiste en reemplazar datos sensibles, como un número de tarjeta de crédito, por un sustituto no sensible llamado token. El token es un código alfanumérico de valor aleatorio e inservible por sí mismo, que representa los datos originales. Este proceso se usa en seguridad de datos para proteger información confidencial y en áreas como el procesamiento de pagos para simplificar las transacciones.

234.LEMATIZACIÓN

Técnicas de preprocesamiento de textos en el procesamiento del lenguaje natural (PLN).

235.WORD EMBEDDINGS

Método utilizado en el procesamiento del lenguaje natural para representar palabras o documentos como vectores numéricos.

236.WORD2VEC

Técnica para el procesamiento de lenguaje natural publicada en 2013. El algoritmo Word2Vec utiliza un modelo de red neuronal para aprender asociaciones de palabras a partir de un gran corpus de texto.

237.GLOVE

(Vectores globales para la representación de palabras) es un modelo de incorporación de palabras diseñado para capturar información estadística global sobre patrones de co-ocurrencia de palabras en un cuerpo.

238.TRANSFORMERS

Son un tipo de arquitectura de red neuronal que revoluciona el Procesamiento del Lenguaje Natural (PLN) y la Visión por Computador al utilizar un mecanismo de autoatención para procesar secuencias de datos.

239.BERT

Técnica basada en redes neuronales para el preentrenamiento del procesamiento del lenguaje. Puede utilizarse para ayudar a discernir el contexto de las palabras en las consultas de búsqueda.

240.GPT

Transformador Generativo Preentrenado, un modelo de IA desarrollado por OpenAI para generar texto y otras formas de contenido similar al humano, y Tabla de Particiones GUID, un estándar más moderno para estructurar y organizar las particiones en un disco duro, que reemplaza al antiguo MBR (Registro de Arranque Maestro).

241.CHATBOT

Es un software que simula conversaciones humanas mediante texto o voz, utilizando técnicas como el Procesamiento del Lenguaje Natural (PLN) para entender al usuario y ofrecer respuestas de forma automática o transferirlo a un agente humano. Los chatbots se usan para automatizar tareas de servicio al cliente, ventas y soporte técnico, mejorando la comunicación y disponibilidad de las empresas.

242.VISIÓN POR COMPUTADORA

Es un campo de la inteligencia artificial que permite a las computadoras "ver" e interpretar imágenes, videos y otros datos visuales, similar a como lo hacen los humanos. Utiliza algoritmos de aprendizaje automático y profundo para analizar estos datos, identificar objetos, patrones y extraer información significativa, impulsando aplicaciones en áreas como la salud, la seguridad, la manufactura y los sistemas autónomos.

243.RECONOCIMIENTO FACIAL

Tecnología biométrica que utiliza inteligencia artificial y algoritmos de aprendizaje automático para identificar o verificar la identidad de una persona analizando los rasgos únicos de su rostro.

244.DETECCIÓN DE OBJETOS

Técnica de visión artificial que permite a los sistemas identificar y localizar objetos específicos dentro de imágenes o videos.

245.RECONOCIMIENTO DE VOZ

Tecnología que permite a los ordenadores y dispositivos entender y procesar el lenguaje humano hablado, convirtiendo la voz en texto o comandos que la máquina puede comprender y ejecutar.

246.CICLO DE VIDA DEL SOFTWARE

Proceso estructurado que abarca todas las etapas de un proyecto de software, desde la concepción de la idea hasta el retiro final del producto, asegurando la calidad, la eficiencia y la gestión de riesgos a través de fases como planificación, análisis de requisitos, diseño, desarrollo, pruebas, implementación y mantenimiento.

247.REQUISITOS FUNCIONALES

Definen el comportamiento y las funciones específicas que debe realizar un sistema de software para satisfacer las necesidades del usuario y los objetivos del negocio.

248.DISEÑO DE SOFTWARE

Proceso de planificar y crear la estructura de un sistema o aplicación de software, que actúa como un "plano" para su posterior desarrollo.

249.ARQUITECTURA DE SOFTWARE

Diseño estructural de un sistema informático, definiendo sus componentes, sus relaciones y las reglas que rigen su comportamiento global.

250.ARQUITECTURA MONOLÍTICA

Modelo de software donde todos los componentes de una aplicación (interfaz de usuario, lógica de negocio y acceso a datos) están integrados en una única unidad cohesiva o en un solo código base.

251.ARQUITECTURA DE MICROSERVICIOS

Enfoque de desarrollo de software que divide una aplicación grande y compleja en servicios pequeños, independientes y de fácil mantenimiento, cada uno con su propio ámbito de responsabilidad y tecnología.

252.ARQUITECTURA ORIENTADA A SERVICIOS

Método de desarrollo de software que estructura aplicaciones como un conjunto de servicios modulares, autónomos y reutilizables que se comunican a través de interfaces estándar.

253.API

Conjunto de reglas, protocolos y herramientas que permite a diferentes programas o aplicaciones comunicarse e intercambiar datos o funcionalidades entre sí de manera segura y estandarizada.

254.REST

Es un estilo arquitectónico que define un conjunto de principios y restricciones para diseñar APIs (Interfaces de Programación de Aplicaciones) web, promoviendo la escalabilidad, eficiencia e independencia entre cliente y servidor.

255.GRAPHQL

Lenguaje de consulta de código abierto para APIs que permite a los clientes solicitar datos específicos y solo los que necesitan, evitando la sobrecarga de datos que a veces ocurre con las APIs REST tradicionales.

256.GRPC

Framework (entorno de trabajo) de Llamada a Procedimiento Remoto (RPC) de código abierto, multiplataforma y de alto rendimiento, desarrollado por Google y ahora gestionado por la Cloud Native Computing Foundation (CNCF).

257.SOAP

Es un protocolo estándar basado en XML que se utiliza para intercambiar información estructurada y permitir que diferentes aplicaciones se comuniquen de forma estandarizada, independientemente de la plataforma o el lenguaje de programación.

258.SDK

Conjunto de herramientas, bibliotecas, documentación y ejemplos de código que los desarrolladores usan para crear aplicaciones que funcionen en una plataforma o sistema operativo específico, como iOS o Windows, o para integrar servicios de terceros, como las funcionalidades de un juego o una red publicitaria.

259.IDE

Entorno de Desarrollo Integrado (Integrated Development Environment). Es una aplicación de software que reúne todas las herramientas necesarias para el desarrollo de software (como editores de código, compiladores, y depuradores) en una única interfaz gráfica, aumentando la productividad del programador al facilitar la escritura, prueba y gestión de código de manera eficiente.

260.DEPURACIÓN (DEBUGGING)

Proceso de identificar y corregir errores o "bugs" en el código de un software o hardware para asegurar que funcione correctamente y eficientemente.

261.TESTING UNITARIO

Es la práctica de verificar que la unidad funcional más pequeña de un código (como una función o un método) funcione correctamente de manera aislada.

262.TESTING DE INTEGRACIÓN

Fase del proceso de pruebas de software donde se unen y prueban varios módulos o componentes de una aplicación para verificar que funcionen correctamente juntos, se comuniquen entre sí e interactúen según lo esperado para lograr las funcionalidades deseadas.

263.TESTING DE SISTEMAS

Etapa de prueba integral que se realiza a un software completamente integrado para verificar que cumple con todos los requisitos funcionales y no funcionales, evaluando su rendimiento y comportamiento bajo condiciones similares a las de un usuario real antes de su lanzamiento.

264.TESTING DE ACEPTACIÓN

Fase final del desarrollo de software donde se verifica formalmente si un sistema cumple con los requisitos del negocio y las necesidades del usuario antes de su lanzamiento, a menudo realizadas por usuarios finales en escenarios del mundo real.

265.MOCKING

Técnica de desarrollo de software que consiste en crear objetos simulados (o "mocks") para reemplazar componentes o dependencias reales en un programa, permitiendo aislar y probar la unidad de código que se está desarrollando sin depender de servicios externos como bases de datos o APIs.

266.CI/CD (INTEGRACIÓN CONTINUA / DESPLIEGUE CONTINUO)

Es una práctica de desarrollo de software que automatiza las etapas de compilación, pruebas y despliegue de código para entregar software de forma frecuente, rápida y confiable.

267.DEVOPS

Combinación de cultura organizacional, prácticas y herramientas que busca unir a los equipos de desarrollo (Dev) y operaciones (Ops) para entregar software de manera más rápida, fiable y eficiente.

268.DOCKER

Plataforma de código abierto para desarrollar y desplegar aplicaciones de manera eficiente, que utiliza contenedores para empaquetar el código y todas sus dependencias en unidades estandarizadas y portátiles.

269.KUBERNETES

Plataforma de código abierto para automatizar el despliegue, la escalabilidad y la administración de aplicaciones en contenedores. Desarrollado inicialmente por Google, Kubernetes, también conocido como "k8s", orquesta los contenedores (microservicios empaquetados con sus dependencias) para asegurar que las aplicaciones funcionen de manera confiable en cualquier infraestructura, ya sea en la nube o en servidores locales.

270.CONTENEDOR

Paquete de software ligero y autocontenido que incluye una aplicación y todas sus dependencias necesarias (código, bibliotecas, configuraciones y archivos) para ejecutarse de manera consistente en cualquier entorno, virtualizando el sistema operativo para proporcionar aislamiento y portabilidad.

271.ORQUESTACIÓN

Coordinación y gestión automatizada de múltiples sistemas, aplicaciones o servicios para ejecutar un proceso o flujo de trabajo complejo, integrando diversas tareas y asegurando que funcionen conjuntamente de forma eficiente.

272.VIRTUALIZACIÓN

Tecnología que consiste en crear copias virtuales de recursos informáticos físicos (como servidores, sistemas operativos o almacenamiento) utilizando software especializado, permitiendo que múltiples entornos virtuales y aislados funcionen simultáneamente en un solo sistema físico.

273.HYPERVISOR

Es un software que actúa como un administrador de recursos informáticos, permitiendo que múltiples máquinas virtuales (VMs) operen de forma independiente en un único hardware físico, como un servidor.

274.QA (QUALITY ASSURANCE)

Conjunto de procesos y prácticas diseñados para prevenir errores y asegurar que un software cumpla con los requisitos de calidad definidos y sea fiable, funcional y seguro para el usuario final.

275.TDD (TEST DRIVEN DEVELOPMENT)

Es una práctica de desarrollo de software donde se escribe una prueba unitaria antes de escribir el código de producción. El proceso sigue un ciclo Rojo-Verde-Refactor: se escribe una prueba que falla (Rojo), se escribe la mínima cantidad de código necesario para que pase (Verde), y luego se limpia y mejora el código sin cambiar su comportamiento (Refactor).

276.BDD (BEHAVIOR DRIVEN DEVELOPMENT)

Desarrollo Guiado por Comportamiento, es una metodología de desarrollo de software ágil que se basa en la colaboración entre desarrolladores, testers y partes interesadas del negocio para definir el comportamiento esperado de una aplicación a través de un lenguaje natural legible para todos.

277.REFACTORIZACIÓN

Proceso de reestructurar el código existente para mejorar su diseño interno, haciéndolo más fácil de entender, mantener y extender, sin alterar su comportamiento o funcionalidad externa.

278.CODE REVIEW

Proceso donde desarrolladores revisan el código escrito por sus compañeros antes de que sea integrado en el proyecto principal, con el fin de encontrar errores, asegurar que cumple con los estándares de calidad, y compartir conocimientos y mejores prácticas dentro del equipo.

279.CONTROL DE VERSIONES

Es una práctica y el uso de sistemas de software (como Git) que registran y gestionan los cambios realizados en un conjunto de archivos, como código fuente o documentos, a lo largo del tiempo. Permite recuperar versiones anteriores, comparar modificaciones, solucionar conflictos y facilita la colaboración entre desarrolladores, protegiendo el trabajo de pérdidas y errores humanos.

280.GIT

Es un sistema de control de versiones distribuido gratuito y de código abierto que permite a los desarrolladores llevar un registro de los cambios en el código fuente de un proyecto de software, facilitando la colaboración simultánea en equipo, el seguimiento del historial de modificaciones, la reversión a versiones anteriores y la gestión eficiente del código.

281.GITHUB

Es una plataforma web en la nube que permite a los desarrolladores almacenar, compartir y colaborar en proyectos de código utilizando el sistema de control de versiones Git.

282.GITLAB

Es una plataforma de DevSecOps todo en uno que proporciona repositorios de código, herramientas de colaboración, automatización de la integración y despliegue continuo (CI/CD) y gestión de proyectos para equipos de desarrollo de software.

283.BITBUCKET

Plataforma de alojamiento de código fuente y colaboración basada en Git propiedad de Atlassian, diseñada para que los equipos de desarrollo de software trabajen de manera eficiente, gestionando sus repositorios, automatizando tareas de integración y despliegue continuo (CI/CD), y colaborando con herramientas como Jira y Trello. Permite a los equipos controlar la calidad del código, monitorear vulnerabilidades de seguridad y automatizar flujos de trabajo para un desarrollo más rápido y seguro.

284.BRANCH

Es una versión paralela y aislada de un proyecto o código fuente, que permite a los desarrolladores trabajar en nuevas funciones o experimentar sin afectar la línea principal de desarrollo.

285.MERGE

Es el acto de combinar dos o más elementos, como datos, código o archivos, para crear un solo conjunto coherente, ya sea para ordenar listas (Merge Sort), integrar cambios de código en sistemas de control de versiones como Git (Git Merge y Merge Request), o combinar tablas en el análisis de datos (como con Pandas en Python).

286.PULL REQUEST

Solicitud que un desarrollador hace a un equipo para que revisen y aprueben cambios realizados en una rama del código, antes de fusionarlos a la rama principal del proyecto.

287.COMMIT

Es un punto de guardado permanente de los cambios en un proyecto, especialmente en sistemas de control de versiones como Git. Cada commit captura una "instantánea" de tu código en un momento específico, creando una versión "segura" del proyecto que puedes consultar o revertir si es necesario.

288.TAG

Palabra clave o etiqueta que se adjunta a información, datos o contenido para organizarlos, clasificarlos y facilitar su identificación y búsqueda.

289.RELEASE

Es una versión específica de un software, producto o conjunto de cambios que se ha completado, probado y puesto a disposición para su uso o instalación por los usuarios finales.

290.DEPLOYMENT

Proceso de trasladar una aplicación o software desde su entorno de desarrollo (donde se crea y prueba) hasta el entorno de producción, haciéndola accesible para los usuarios finales.

291.ROLLBACK

Proceso de revertir o deshacer cambios realizados en un sistema, software o base de datos, restaurando a un estado previo y conocido, generalmente como respuesta a un error o para recuperar la estabilidad.

292.FEATURE FLAG

Técnica de desarrollo de software que permite activar o desactivar funcionalidades específicas en tiempo real, sin necesidad de modificar el código fuente o realizar un nuevo despliegue.

293.DOCUMENTACIÓN TÉCNICA

Conjunto de guías, especificaciones y explicaciones escritas que describen cómo se diseña, desarrolla, utiliza y mantiene un sistema, software o producto de tecnología, sirviendo como puente entre la complejidad técnica y los diferentes tipos de usuarios o desarrolladores.

294.MANUAL DEL USUARIO

Documento de referencia que proporciona instrucciones detalladas y guías sobre cómo instalar, operar, configurar y solucionar problemas de un software o sistema informático de manera eficaz.

295.BASE DE DATOS

Colección organizada de datos almacenados electrónicamente en un sistema informático, que permite a los usuarios recopilar, gestionar, actualizar y recuperar información de manera eficiente.

296.BASE DE DATOS RELACIONAL

Organiza la información en tablas con filas y columnas, como hojas de cálculo, permitiendo establecer conexiones entre los datos de diferentes tablas mediante identificadores comunes (claves).

297.BASE DE DATOS NO RELACIONAL (NOSQL)

Almacena y maneja datos en formatos distintos a las tablas tradicionales, como en documentos, pares clave-valor o grafos, en lugar de las filas y columnas de las bases de datos SQL.

298.SQL

Es un lenguaje de programación utilizado para gestionar y manipular datos en bases de datos relacionales. Sirve para almacenar, recuperar, actualizar y eliminar información, así como para administrar el rendimiento de la base de datos.

299.MYSQL

Es un sistema de gestión de bases de datos relacionales (SGBDR) de código abierto, lo que significa que su código es público y se puede usar, modificar y distribuir de forma gratuita.

300.POSTGRESQL

Sistema de gestión de bases de datos relacionales (RDBMS) de código abierto, conocido por su robustez, fiabilidad y alto rendimiento, que permite organizar y gestionar grandes volúmenes de datos.

301.ORACLE DATABASE

Es un Sistema de Gestión de Bases de Datos Relacionales (RDBMS) desarrollado por Oracle Corporation para almacenar, gestionar y recuperar datos de forma organizada.

302.MICROSOFT SQL SERVER

Es un sistema de gestión de bases de datos relacionales (RDBMS) de Microsoft que almacena y recupera datos según lo solicitado por otras aplicaciones.

303.MARIADB

Sistema de Gestión de Bases de Datos Relacionales (RDBMS) gratuito y de código abierto, creado por los desarrolladores originales de MySQL debido a preocupaciones por la comercialización de MySQL tras su adquisición por Oracle en 2009. Se utiliza para almacenar y gestionar datos de forma estructurada en tablas, siendo compatible con la mayoría de las aplicaciones que usan MySQL y ofreciendo mejoras en rendimiento y funcionalidades, incluyendo motores de almacenamiento y características similares a las bases de datos NoSQL.

304.SQLITE

Motor de base de datos relacional sin servidor, ligero y de código abierto que almacena la base de datos completa en un solo archivo, lo que lo hace ideal para aplicaciones integradas, móviles y de escritorio con requisitos de datos reducidos. No necesita un proceso de servidor independiente, ya que es una biblioteca que se integra directamente en el programa de la aplicación, ofreciendo acceso a datos de manera eficiente y simple.

305.MONGODB

Base de datos NoSQL de código abierto que almacena datos en formato de documentos BSON (similar a JSON) en lugar de tablas y filas tradicionales. Su principal ventaja es la flexibilidad del esquema, que permite variar la estructura de los datos y adaptar el sistema a datos semiestructurados y no estructurados, ideal para la escalabilidad y la agilidad en el desarrollo de aplicaciones modernas.

306.CASSANDRA

Cassandra es una base de datos NoSQL de código abierto, diseñada para manejar grandes volúmenes de datos en múltiples servidores, ofreciendo alta disponibilidad y escalabilidad horizontal para aplicaciones como IoT y análisis en tiempo real.

307.REDIS

Servidor de diccionario remoto (REmote DIctionary Server) de código abierto, un almacén de datos en memoria que funciona como base de datos NoSQL de clave-valor, sistema de caché distribuida y broker de mensajería.

308.NEO4J

Base de datos orientada a grafos (BDOG) que almacena y procesa datos utilizando la estructura de un grafo, compuesto por nodos (que representan entidades) y aristas (que representan las relaciones entre ellos).

309.COUCHDB

Base de datos documental NoSQL de código abierto, desarrollada por la Apache Software Foundation, que almacena datos en documentos en formato JSON en lugar de tablas. Se destaca por su facilidad para la replicación de datos entre dispositivos y servidores, el soporte de consultas a través de una API REST y JavaScript, y su capacidad para trabajar con aplicaciones web y móviles.

310.FIRESTORE

Base de datos de documentos NoSQL, escalable y flexible, ofrecida por Google dentro del ecosistema de Firebase. Permite a los desarrolladores almacenar, sincronizar y consultar datos de forma sencilla y en tiempo real para aplicaciones web y móviles.

311.DYNAMODB

Es un servicio de base de datos NoSQL sin servidor y completamente administrado de Amazon Web Services (AWS). Ofrece escalabilidad, durabilidad y rendimiento rápido y predecible para aplicaciones modernas, como backends de aplicaciones web y móviles, soportando modelos de datos de clave-valor y documentos.

312.TABLA

Estructura bidimensional que organiza datos en filas (registros) y columnas (campos), similar a un formato impreso, pero para el almacenamiento y manejo eficiente de información en sistemas digitales.

313.COLUMNA

Conjunto de celdas o datos organizados verticalmente en una tabla, como en hojas de cálculo o bases de datos. Se identifican mediante letras en su cabecera (por ejemplo, A, B, C en Excel) y contienen datos del mismo tipo para todos sus elementos.

314.FILA (REGISTRO)

Representa una única entrada o instancia de datos dentro de una tabla, agrupando varios campos (que son como las columnas) que definen un objeto o entidad de manera completa, como los datos de un cliente específico.

315.CLAVE PRIMARIA

Columna o conjunto de columnas en una tabla de base de datos que identifica de forma única cada fila (registro) de esa tabla.

316.CLAVE FORÁNEA

Columna o un conjunto de columnas en una tabla que establece un vínculo con la clave primaria (o una clave única) de otra tabla, o incluso de la misma tabla.

317.ÍNDICE

Estructura de datos que acelera la búsqueda y recuperación de información, ya sea para un motor de búsqueda, una base de datos, un procesador de texto o un programa informático. Funciona creando un orden de punteros o referencias basado en valores o claves, permitiendo un acceso más rápido a los datos en comparación con el escaneo completo de un conjunto de información.

318.VISTA

Es principalmente una tabla virtual o una consulta guardada que presenta los datos de una o varias tablas subyacentes como si fuera una tabla real, pero sin almacenar los datos en sí.

319.TRIGGER

Procedimiento almacenado en una base de datos que se ejecuta automáticamente cuando ocurre un evento específico, como una operación de inserción (INSERT), actualización (UPDATE) o eliminación (DELETE) de datos en una tabla.

320.PROCEDIMIENTO

Secuencia de instrucciones predefinidas almacenadas en una base de datos que se pueden ejecutar para realizar tareas específicas de forma repetitiva y eficiente.

321.ALMACENAMIENTO

Proceso de guardar y conservar datos digitales en dispositivos y sistemas para su posterior recuperación y uso, abarcando desde el almacenamiento en discos duros y la nube hasta la organización de la información en estructuras lógicas.

322.TRANSACCIÓN

Conjunto de una o más operaciones (lectura, escritura, modificación) que se realizan como una unidad lógica de trabajo indivisible, lo que significa que o se completan todas exitosamente o ninguna se completa, asegurando así la integridad y consistencia de los datos.

323.ACID

Es un acrónimo que describe las cuatro propiedades que garantizan la fiabilidad de las transacciones en bases de datos: Atomicidad, Consistencia, Aislamiento y Durabilidad.

324.NORMALIZACIÓN

Proceso de organizar los datos en una base de datos para minimizar la redundancia y la dependencia de datos, mejorando así la integridad de la información. Esto se logra dividiendo tablas grandes en tablas más pequeñas y relacionadas, siguiendo un conjunto de reglas llamadas formas normales (como 1NF, 2NF, 3NF), que también ayudan a prevenir anomalías de inserción, actualización y eliminación de datos.

325.DESNORMALIZACIÓN

Técnica que consiste en introducir intencionalmente redundancia de datos en una base de datos para optimizar el rendimiento de lectura.

326.ESQUEMA

Estructura, plano o plantilla que define la organización, composición y las relaciones de la información o de un sistema.

327.MODELO ENTIDAD - RELACIONADA

Herramienta gráfica utilizada para diseñar y entender la estructura de una base de datos, representando los datos como entidades (cosas, personas o conceptos), sus atributos (características) y las relaciones entre ellas.

328.DIAGRAMA ER

Representación visual de la estructura de una base de datos, que muestra cómo se relacionan diferentes elementos o "entidades" (como clientes, productos, o pedidos) y sus atributos (características), permitiendo diseñar y entender la organización de los datos en un sistema. Es una herramienta fundamental para analistas y diseñadores de bases de datos.

329.CARDINALIDAD

Describe el número de instancias en una tabla que pueden relacionarse con el número de instancias en otra tabla, especificando cómo se vinculan las entidades en un modelo de datos, como uno a uno, uno a varios o varios a varios.

330.QUERY

Solicitud o consulta que se envía a una base de datos o un sistema de información para recuperar, modificar o manipular datos.

331.SELECT

Es la palabra clave fundamental en el lenguaje SQL para recuperar y mostrar datos de una o más tablas de una base de datos. También se refiere al elemento <select> en HTML, que crea un menú desplegable de opciones para el usuario en una página web.

332.INSERT

Acción de añadir nuevos datos a una estructura preexistente, como agregar registros a una base de datos o elementos a una lista.

333.UPDATE

Proceso de reemplazar una versión anterior de un software o sistema con una nueva versión que corrige problemas, añade funciones o mejora el rendimiento.

334.DELETE

Es una acción que puede referirse a la tecla física de un teclado que elimina caracteres o archivos, o a una instrucción de software para remover datos, como la instrucción SQL DELETE para borrar registros en bases de datos, o el método HTTP DELETE en APIs para eliminar recursos.

335.JOIN

Operación de unir o combinar elementos de diferentes conjuntos de datos para obtener un resultado unificado. El contexto más común para el término es en bases de datos relacionales, donde la sentencia JOIN en SQL combina filas de dos o más tablas basándose en una columna relacionada para crear un único conjunto de resultados.

336.INNER JOIN

(O unión interna) es una operación en SQL que combina registros de dos o más tablas basándose en una condición de coincidencia entre columnas comunes a esas tablas.

337.LEFT JOIN

Operación que combina dos tablas (la izquierda y la derecha) y devuelve todos los registros de la tabla de la izquierda, junto con los registros coincidentes de la tabla de la derecha.

338.RIGHT JOIN

Es una operación de SQL que une dos o más tablas y devuelve todas las filas de la tabla "derecha" (la segunda especificada), y las filas coincidentes de la tabla "izquierda" (la primera especificada), rellenando con NULL los valores de las columnas de la tabla izquierda cuando no hay una coincidencia.

339.FULL JOIN

Operación que combina registros de dos tablas y devuelve todos los registros de ambas tablas, incluyendo tanto las filas coincidentes como las que no tienen correspondencia en la otra tabla, rellenando con NULL en los campos donde no hay datos.

340.SUBQUERY

Consulta SQL anidada dentro de otra, también conocida como consulta interna o consulta secundaria.

341.AGRUPACIÓN (GROUP BY)

Se utiliza para agrupar filas que tienen valores idénticos en una o más columnas, y luego aplicar funciones de agregación (como SUM, COUNT, AVG) para realizar cálculos resumen sobre cada grupo. Su función principal es condensar datos, permitiendo obtener información agregada de manera eficiente, como el total de ventas por región o la cantidad de estudiantes por asignatura.

342.HAVING

La cláusula de SQL utilizada para filtrar los resultados de una consulta después de que se ha aplicado la cláusula GROUP BY y se han realizado funciones de agregación como COUNT, SUM o AVG.

343.FUNCIONES DE AGREGACIÓN

Función que toma un conjunto de valores y calcula un único valor escalar para representar ese conjunto, como la suma (SUM), el promedio (AVG), el máximo (MAX) o el mínimo (MIN) de una columna.

344.REPLICACIONES DE BASES DE DATOS

Proceso por el cual se copian y sincronizan los datos de una base de datos entre múltiples servidores o ubicaciones, permitiendo mayor disponibilidad, tolerancia a fallos, balanceo de carga y respaldo de información crítica.

345.SHARDING

Técnica para dividir horizontalmente una base de datos o una red blockchain grande en partes más pequeñas, llamadas shards o fragmentos, que son manejables y se distribuyen en múltiples servidores.

346.COMPUTACIÓN EN LA NUBE

Modelo que permite acceder a recursos informáticos (como servidores, almacenamiento, bases de datos, software, redes y análisis) a través de Internet, en lugar de tenerlos físicamente en su propia infraestructura.

347.NUBE PÚBLICA

Modelo de informática en la nube donde un proveedor externo pone recursos informáticos (servidores, almacenamiento, aplicaciones, etc.) a disposición de múltiples clientes a través de Internet.

348.NUBE PRIVADA

Entorno de computación en la nube diseñado y dedicado exclusivamente para una sola organización, ofreciéndole control total sobre sus recursos informáticos, mayor seguridad y la capacidad de personalizar el entorno según sus necesidades específicas.

349.NUBE HÍBRIDA

Modelo de infraestructura en la nube que combina servicios de nubes públicas y privadas, permitiendo compartir datos y aplicaciones entre ambas, brindando mayor flexibilidad, escalabilidad y opciones de despliegue según las necesidades del negocio.

350.MULTCLOUD

Es un servicio en la nube que permite a los usuarios conectar y gestionar múltiples cuentas de diferentes servicios de almacenamiento en la nube (como Google Drive, Dropbox, OneDrive, etc.) desde una única plataforma, facilitando la transferencia de archivos, la copia de seguridad y la sincronización entre ellas sin necesidad de descargar y volver a subir datos.

351.SAAS (SOFTWARE AS A SERVICE)

Modelo de distribución de software basado en la nube que permite a los usuarios acceder a aplicaciones a través de internet mediante una suscripción, en lugar de comprar e instalar el software localmente.

352.PAAS (PLATFVICE)ORM

Es un modelo de informática en la nube que provee una plataforma completa (hardware, sistemas operativos, middleware, bases de datos, etc.) a través de internet para que los desarrolladores creen, ejecuten y gestionen aplicaciones, sin tener que encargarse de la infraestructura subyacente.

353.IAAS (INFRASTRUCTURE AS A SERVICE)

Modelo de computación en la nube donde se alquilan recursos de TI, como servidores, almacenamiento y redes, a través de Internet.

354.FAAS (FUNCTION AS A SERVICE)

Modelo de computación en la nube sin servidor que permite a los desarrolladores ejecutar código en respuesta a eventos, sin preocuparse por la gestión de la infraestructura subyacente.

355.SERVERLESS

Es un modelo de computación en la nube donde los proveedores de servicios en la nube administran y aprovisionan automáticamente la infraestructura de servidores necesaria para ejecutar el código, liberando a los desarrolladores de tener que gestionar servidores, ya que se centran únicamente en la lógica de la aplicación y pagan solo por los recursos que realmente usan.

356.CLOUD-NATIVE

Enfoque para construir y ejecutar aplicaciones que aprovechan al máximo el modelo de la nube, dividiendo las aplicaciones en microservicios y empaquetándolos en contenedores gestionados por orquestadores dinámicos.

357.CONTENEDORES EN LA NUBE

Paquetes de software portátiles y auto-suficientes que incluyen todo lo necesario (código, bibliotecas y dependencias) para ejecutar una aplicación en la nube.

358.ESCALABILIDAD HORIZONTAL

Capacidad de un sistema para manejar una carga de trabajo creciente añadiendo más recursos (como servidores o nodos) y distribuyendo la carga de trabajo entre ellos, en lugar de mejorar la capacidad de una sola máquina.

359.ESCALABILIDAD VERTICAL

Es la práctica de aumentar los recursos de hardware de un sistema o máquina individual para mejorar su capacidad de rendimiento y manejar cargas de trabajo más pesadas, como más potencia de procesamiento (CPU), más memoria RAM o mayor almacenamiento.

360.ELASTICIDAD

Capacidad de un sistema, especialmente en la nube, de ajustar su capacidad de procesamiento, memoria y almacenamiento de forma automática y rápida para adaptarse a las fluctuaciones de la demanda.

361.BALANCEO DE CARGA

Es la práctica de distribuir el tráfico de internet y las solicitudes entre varios servidores para optimizar el rendimiento, aumentar la disponibilidad y la escalabilidad, y minimizar la latencia.

362.ALTA DISPONIBILIDAD

Capacidad de un sistema, aplicación o dato para operar de manera continua y confiable, asegurando que esté accesible para los usuarios autorizados incluso ante fallos o interrupciones no planificadas.

363.RECUPERACIÓN DE DESASTRES (DR)

Proceso y conjunto de estrategias, tecnologías y procedimientos que una organización implementa para restablecer su infraestructura y operaciones de TI después de un evento catastrófico, como desastres naturales, fallas de hardware o ciberataques.

364.REQUISITOS NO FUNCIONALES

Son las propiedades y atributos de calidad que un sistema de software debe cumplir, como el rendimiento, la seguridad, la usabilidad y la fiabilidad, en lugar de las funciones específicas que realiza el sistema.

365.CLOUD BURSTING

Estrategia de computación en la nube en la que una aplicación que normalmente se ejecuta en una nube privada o centro de datos se expande automáticamente a una nube pública cuando se alcanza la capacidad máxima, para manejar picos de demanda.

366.EDGE COMPUTING

Modelo informático distribuido que acerca el procesamiento de datos y el almacenamiento a los dispositivos y usuarios que generan la información, en lugar de enviarla a un centro de datos centralizado (la nube).

367.FOG COMPUTING

Arquitectura de red que acerca las capacidades de procesamiento y almacenamiento de datos de la nube hacia el borde de la red, es decir, más cerca de donde se generan los datos. Actúa como un intermediario entre los dispositivos y la nube central, reduciendo la latencia, el consumo de ancho de banda y permitiendo una toma de decisiones más rápida, especialmente en entornos como el Internet de las Cosas (IoT).

368.AWS (AMAZON WEB SERVICES)

Plataforma líder mundial de computación en la nube que ofrece más de 200 servicios integrales de centros de datos para organizaciones de todos los tamaños. Proporciona IaaS, PaaS y SaaS, incluyendo potencia de procesamiento virtual, almacenamiento, bases de datos y redes, permitiendo a los clientes reducir costos, ser más ágiles e innovar más rápido.

369.MICROSOFT AZURE

Plataforma de servicios de computación en la nube que permite a las empresas crear, implementar y gestionar aplicaciones y servicios a través de Internet, utilizando recursos como procesamiento, almacenamiento, redes y bases de datos.

370.GOOGLE CLOUD PLATFORM

Conjunto de servicios de computación en la nube ofrecidos por Google que permite a empresas y desarrolladores crear, implementar y escalar aplicaciones, almacenar datos y realizar análisis.

371.IBM CLOUD

Plataforma de computación en la nube de IBM que ofrece servicios de IaaS, PaaS y SaaS para empresas, con soluciones híbridas y de inteligencia artificial.

372.ORACLE CLOUD

Plataforma de servicios en la nube de Oracle que ofrece recursos informáticos bajo demanda (servidores, almacenamiento, redes) y servicios empresariales como bases de datos, análisis, seguridad e integración.

373.ALIBABA CLOUD

Plataforma de servicios en la nube de Alibaba Group que ofrece recursos informáticos bajo demanda, almacenamiento, bases de datos, inteligencia artificial y servicios empresariales en línea.

374.KUBERNETES EN LA NUBE

Orquestación y gestión de aplicaciones en contenedores mediante servicios como Amazon EKS, facilitando la implementación y escalado de clústeres Kubernetes sin gestionar directamente la infraestructura.

375.TERRAFORM

Herramienta de código abierto de HashiCorp que permite definir y gestionar infraestructura en la nube como código (IaC), usando HCL (HashiCorp Configuration Language) para aprovisionar recursos de forma automatizada y reproducible.

376.ANSIBLE

Plataforma de automatización de código abierto para aprovisionamiento de sistemas, gestión de configuraciones y despliegue de aplicaciones, que utiliza archivos YAML simples para definir tareas sin necesidad de agentes.

377.CLOUDFORMATION

Servicio de AWS que permite definir y aprovisionar infraestructura en la nube mediante plantillas en formato JSON o YAML. Automatiza el ciclo de vida completo de los recursos usando el enfoque de Infraestructura como Código.

378.OPENSTACK

Plataforma de código abierto para construir y administrar nubes privadas y públicas. Proporciona servicios de computación, almacenamiento y redes como IaaS, con APIs abiertas y personalización flexible.

379.CLOUD FOUNDRY

Plataforma como servicio (PaaS) de código abierto que permite a los desarrolladores implementar y escalar aplicaciones en la nube fácilmente, sin preocuparse por la infraestructura subyacente.

380.CDN (CONTENT DELIVERY NETWORK)

Red de servidores distribuidos geográficamente que entrega contenido web como imágenes, videos y archivos de forma rápida y eficiente, reduciendo la latencia al acercar los datos al usuario final.

381.CLOUD STORAGE

Modelo de almacenamiento donde los datos se guardan en servidores remotos administrados por un proveedor de servicios en la nube, accesibles desde cualquier lugar vía Internet.

382.BLOB STORAGE

Servicio de almacenamiento en la nube para grandes volúmenes de datos no estructurados o binarios, como imágenes, audio, video y archivos de respaldo. Común en plataformas como Azure.

383.OBJECT STORAGE

Modelo de almacenamiento que organiza los datos como objetos con metadatos, ideal para escalabilidad y recuperación eficiente. Utilizado por servicios como Amazon S3 y Google Cloud Storage.

384.FILE STORAGE

Sistema de almacenamiento que organiza los datos en archivos y carpetas jerárquicas, accesibles mediante rutas específicas, como en sistemas NAS o servicios cloud basados en archivos.

385.BLOCK STORAGE

Tecnología que divide los datos en bloques con identificadores únicos. Es ideal para bases de datos y aplicaciones críticas por su alto rendimiento y acceso directo a cada bloque.

386.S3 (SIMPLE STORAGE SERVICE)

Servicio de almacenamiento de objetos de AWS que permite almacenar y recuperar cualquier cantidad de datos en cualquier momento y desde cualquier lugar a través de Internet.

387.GLACIER (ALMACENAMIENTO FRÍO)

Servicio de almacenamiento de bajo costo en AWS diseñado para archivado de datos y copias de seguridad a largo plazo. Ofrece acceso infrecuente y recuperación más lenta.

388.COLOCATION

Práctica en la que una empresa alquila espacio físico y recursos en un centro de datos de terceros para instalar y operar sus propios servidores y equipos.

389.API GATEWAY

Punto de entrada único para gestionar, proteger y monitorizar las llamadas a APIs, actuando como intermediario entre clientes y servicios backend. Permite autenticación, control de tráfico y logging.

390.CLOUD SECURITY

Conjunto de tecnologías, prácticas y políticas destinadas a proteger los sistemas, aplicaciones y datos que se encuentran en entornos de computación en la nube contra amenazas y accesos no autorizados.

391.ZERO TRUST EN LA NUBE

Modelo de seguridad que asume que no se debe confiar automáticamente en ningún usuario, dispositivo o red, y requiere verificación estricta antes de conceder acceso a recursos, tanto dentro como fuera de la nube.

392.IAM (IDENTITY AND ACCESS MANAGEMENT)

Disciplina de ciberseguridad que define y gestiona los derechos de acceso de los usuarios a recursos de TI, como datos, aplicaciones y redes. Asegura que las personas correctas tengan los permisos adecuados para acceder a lo necesario, mientras se impide el acceso no autorizado.

393.MULTI-TENANCY

Arquitectura de software donde una única instancia de una aplicación y su infraestructura sirve a múltiples clientes o "inquilinos" (tenants), manteniendo sus datos y configuraciones aislados entre sí.

394.FACTURACIÓN EN LA NUBE

Sistema que permite generar, enviar, almacenar y gestionar facturas electrónicas mediante plataformas digitales accesibles por Internet, aprovechando infraestructura remota en lugar de sistemas locales.

395.SPOT INSTANCES

Recursos computacionales que utilizan capacidad no utilizada de los proveedores de nube a precios reducidos, ideales para cargas de trabajo flexibles o temporales.

396.RESERVED INSTANCES

Modelo de precios en la nube que permite reservar recursos por períodos largos (1 o 3 años) con descuentos respecto a precios bajo demanda, útil para cargas de trabajo constantes.

397.WINDOWS

Familia de sistemas operativos desarrollados por Microsoft, diseñada para gestionar hardware y software en computadoras personales y otros dispositivos.

406.DISTRIBUCIÓN LINUX

407.UBUNTU

Plataforma basada en Linux que ofrece soluciones empresariales como SUSE Linux Enterprise Server (SLES) y openSUSE, utilizadas en entornos locales y en la nube.

413.ARCH LINUX

Distribución de Linux minimalista orientada a usuarios avanzados, conocida por su modelo de actualizaciones continuas (rolling release) y filosofía KISS (Keep It Simple, Stupid).

414.GENTOO

Distribución de Linux gratuita y de código abierto, diseñada para usuarios avanzados, que se caracteriza por su personalización profunda y el uso de un sistema de gestión de paquetes llamado Portage.

415.KERNEL MONOLÍTICO

Arquitectura de sistema operativo donde todos los servicios del sistema operativo (gestión de memoria, procesos, archivos, etc.) se ejecutan en el mismo espacio de direcciones privilegiado (el modo núcleo o kernel).

416.MICROKERNEL

Núcleo de sistema operativo mínimo que implementa las funciones esenciales, como la gestión de memoria y la comunicación entre procesos (IPC), y delega la mayor parte de los servicios a espacios de direcciones de usuario.

417.SISTEMAS DE ARCHIVOS (FILE SYSTEM)

Estructura que utiliza un sistema operativo para organizar, gestionar y almacenar datos en un dispositivo de almacenamiento, como un disco duro o una unidad USB.

418.FAT32

Sistema de archivos antiguo conocido por su amplia compatibilidad. No permite archivos mayores a 4 GB, por lo que es común en memorias USB y tarjetas SD.

419.EXFAT

Sistema de archivos desarrollado por Microsoft, diseñado para almacenamiento flash como tarjetas SDXC. Soporta archivos de gran tamaño y es más eficiente que FAT32.

420.NTFS

Sistema de archivos desarrollado por Microsoft para Windows, que ofrece características avanzadas como permisos, cifrado, cuotas de disco y registro para recuperación ante fallos.

421.EXT4

Sistema de archivos moderno para Linux, sucesor de EXT3, que ofrece mayor rendimiento, escalabilidad y confiabilidad.

422.BTRFS

Sistema de archivos avanzado con enfoque copy-on-write (COW), que permite mantener versiones de archivos, ideal para snapshots y tolerancia a fallos.

423.XFS

Sistema de archivos de alto rendimiento diseñado para manejar archivos muy grandes, con soporte para multihilos, journaling y comprobación de integridad mediante metadatos.

424.HFS+

Sistema de archivos desarrollado por Apple, utilizado en Mac entre 1998 y 2019, antes de ser reemplazado por APFS.

425.APFS

Sistema de archivos moderno de Apple, optimizado para almacenamiento SSD y usado en macOS, iOS, iPadOS y otros dispositivos Apple.

426.SWAP

Área del disco duro utilizada como memoria virtual para extender la RAM física, permitiendo almacenar datos temporalmente cuando la memoria principal está llena.

427.PARTICIÓN

División lógica de un disco físico en secciones independientes que actúan como unidades separadas para organizar datos o instalar distintos sistemas operativos.

428.MBR

(Master Boot Record) Esquema tradicional de particionamiento de discos que permite hasta cuatro particiones primarias y contiene el código de arranque.

429.GPT (GUID PARTITION TABLE)

Esquema moderno de particionamiento de discos que reemplaza al MBR, permite más particiones y soporta discos de mayor tamaño. Utiliza identificadores únicos globales (GUID).

430.SCHEDULER DE CPU

Componente del sistema operativo encargado de asignar el uso de la CPU a los procesos en ejecución, gestionando el orden y duración de su acceso.

431.PLANIFICACIÓN DE PROCESOS

Función del sistema operativo que decide qué procesos deben ejecutarse y en qué orden, optimizando el uso del procesador y asegurando una multitarea eficiente.

432.FIFO (FIRST IN FIRST OUT)

Algoritmo de planificación que ejecuta los procesos en el orden en que llegan. El primero en llegar es el primero en ser atendido.

433.SJF (SHORTEST JOB FIRST)

Algoritmo de planificación de CPU que prioriza los procesos que requieren menos tiempo de ejecución, minimizando el tiempo promedio de espera.

434.PRIORIDAD

Método de asignación de niveles de importancia a procesos, permitiendo que aquellos con mayor prioridad se ejecuten antes que los de menor prioridad.

435.INTERRUPCIÓN

Señal que indica a la CPU que debe detener su tarea actual para atender un evento urgente, como una entrada/salida o error del sistema.

436.KERNEL SPACE

Zona protegida de la memoria del sistema donde se ejecuta el núcleo del sistema operativo y los controladores, con acceso completo al hardware.

437.USER SPACE

Área de memoria donde se ejecutan las aplicaciones de usuario, con acceso restringido a los recursos del sistema para garantizar la seguridad y estabilidad.

438.MODO USUARIO

Estado restringido en el que se ejecutan las aplicaciones de usuario, sin acceso directo al hardware ni a las funciones críticas del sistema operativo.

439.MODO KERNEL

Estado privilegiado en un sistema operativo donde el software tiene acceso ilimitado al hardware y a todos los recursos del sistema. Las funciones críticas del sistema operativo, el kernel y los controladores operan en este modo para gestionar directamente el hardware y la memoria.

440.CONTEXT SWITCH

Proceso mediante el cual el sistema operativo guarda el estado de una tarea (proceso o hilo) y lo restaura más tarde para reanudarla, permitiendo así la ejecución de múltiples tareas en un sistema multitarea.

441.DEADLOCK

Situación en la que dos o más procesos quedan bloqueados permanentemente, esperando mutuamente recursos que nunca se liberan, lo que impide que continúen con su ejecución.

442.CONDICIÓN DE CARRERA

Error de software que ocurre cuando múltiples procesos o hilos acceden simultáneamente a recursos compartidos y el resultado depende del orden de ejecución, provocando comportamientos impredecibles.

443.SEMÁFORO (SISTEMAS OPERATIVOS)

Variable de sincronización que gestiona el acceso concurrente de procesos o hilos a recursos compartidos, evitando conflictos y garantizando la integridad de los datos.

444.MONITOR (CONCURRENCIA)

Construcción de software que combina datos compartidos y procedimientos de sincronización, asegurando que solo un hilo acceda a dichos datos a la vez mediante mecanismos de exclusión mutua.

445.SYSTEM CALL

Interfaz que permite a los programas de usuario solicitar servicios al sistema operativo, como acceso a archivos, memoria o dispositivos.

446.CIENCIA DE DATOS

Campo multidisciplinar que combina estadísticas, programación y conocimiento del dominio para analizar grandes volúmenes de datos y extraer conocimiento valioso.

447.ANALÍTICA DE DATOS

Proceso de examinar datos para obtener información útil que apoye la toma de decisiones, utilizando herramientas estadísticas, matemáticas y de visualización.

448.MINERÍA DE DATOS

Proceso de explorar grandes conjuntos de datos mediante algoritmos analíticos para descubrir patrones ocultos, correlaciones y tendencias.

449.BIG DATA

Conjunto de tecnologías y procesos diseñados para capturar, almacenar y analizar grandes volúmenes de datos que no pueden ser gestionados por métodos tradicionales.

450.DATA LAKE

Repositorio centralizado que permite almacenar datos en su formato original, estructurados y no estructurados, facilitando su análisis posterior sin necesidad de estructuración previa.

451.DATA WAREHOUSE

Sistema centralizado diseñado para almacenar, integrar y analizar datos de múltiples fuentes, permitiendo generar reportes y análisis estratégicos.

452.ETL (EXTRACT, TRANSFORM, LOAD)

Proceso de integración de datos que extrae información de múltiples fuentes, la transforma según reglas de negocio y la carga en un destino como un almacén de datos.

453.ELT

Proceso que primero extrae y carga los datos en su destino final (como un data lake) y luego realiza la transformación directamente en ese entorno.

454.DATA PIPELINE

Secuencia automatizada de procesos para mover, transformar y preparar datos desde su origen hasta su destino, como parte de un flujo de análisis de datos.

455.STREAMING DE DATOS

Procesamiento en tiempo real de flujos continuos de datos conforme se generan, sin necesidad de almacenarlos primero para su análisis.

456.APACHE HADOOP

Framework de código abierto para el almacenamiento y procesamiento distribuido de grandes volúmenes de datos en clústeres de servidores.

457.APACHE SPARK

Motor de procesamiento distribuido que permite realizar análisis de datos masivos de forma rápida, aprovechando la computación en memoria.

458.APACHE FLINK

Motor de procesamiento de datos en tiempo real y por lotes, con alta tolerancia a fallos, gestión eficiente del estado y baja latencia.

459.APACHE STORM

Sistema distribuido y de código abierto que procesa flujos de datos en tiempo real desde múltiples fuentes, como redes sociales o sensores.

460.APACHE KAFKA

Plataforma de eventos distribuida que permite la publicación, almacenamiento y procesamiento de flujos de datos en tiempo real de manera confiable y escalable.

461.MAPREDUCE

Modelo de programación para el procesamiento paralelo de grandes volúmenes de datos que utiliza dos funciones principales: Mapper para transformar y Reducer para agrupar y resumir los resultados.

462.HDFS (HADOOP DISTRIBUTED FILE SYSTEM)

Es un sistema de archivos distribuido y de código abierto, parte del ecosistema de Apache Hadoop, que gestiona grandes volúmenes de datos (Big Data) distribuyéndolos a través de un clúster de máquinas para ofrecer alta disponibilidad, tolerancia a fallos y alto rendimiento.

463.SQOOP

Herramienta de código abierto y línea de comandos para transferir datos de manera eficiente entre bases de datos relacionales (RDBMS) y sistemas de big data como Apache Hadoop.

464.HIVE

Un sistema de almacenamiento de datos para el ecosistema de Apache Hadoop que permite analizar grandes conjuntos de datos (big data) usando un lenguaje de consulta similar a SQL, llamado HiveQL.

465.PIG

Plataforma de alto nivel que simplifica la creación de programas MapReduce para el análisis de grandes conjuntos de datos en Hadoop.

466.PRESTO

Motor de consulta SQL distribuido de código abierto diseñado para analizar grandes volúmenes de datos de diversas fuentes de manera rápida, usando SQL y ejecutándose en paralelo sobre una arquitectura basada en memoria.

467.DATABRICKS

Plataforma de análisis unificado en la nube fundada por los creadores de Apache Spark que simplifica el procesamiento de big data, la ciencia de datos y la inteligencia artificial.

468.SNOWFLAKE

Plataforma de datos nativa de la nube que ofrece un servicio de almacén de datos (data warehouse), lago de datos (data lake) y plataforma para la ciencia de datos y aplicaciones, disponible en múltiples proveedores de nube pública.

469.GOOGLE BIGQUERY

Almacén de datos y plataforma de análisis de big data sin servidor y gestionado por Google que permite a las empresas extraer información valiosa de grandes volúmenes de datos de forma rápida, sencilla y escalable, utilizando SQL y aprendizaje automático integrado.

470.AMAZON REDSHIFT

Almacén de datos en la nube completamente administrado y escalable ofrecido por Amazon Web Services (AWS) que permite analizar grandes volúmenes de datos (a escala de petabytes) de manera rápida y rentable.

471.MICROSOFT SYNAPSE ANALYTICS

Servicio unificado de análisis de datos en la nube que combina almacenamiento de datos empresariales, análisis de big data, exploración de datos e integración de datos en una única plataforma.

472.TABLEAU

Plataforma líder de análisis visual y Business Intelligence (BI) que permite a los usuarios explorar, analizar y visualizar datos de forma rápida e intuitiva para tomar decisiones informadas.

473.POWER BI

Plataforma de Business Intelligence (BI) de Microsoft que permite a los usuarios analizar y visualizar datos, crear paneles interactivos e informes para facilitar la toma de decisiones.

474.QLIKVIEW

Plataforma de Business Intelligence (BI) y análisis que permite crear y explorar aplicaciones y cuadros de mando interactivos para descubrir información y tomar decisiones basadas en datos.

475.GRAFANA

Plataforma de código abierto para la visualización de datos y la monitorización de infraestructuras y aplicaciones.

476.DATA GOVERNANCE

Conjunto de políticas, procesos, roles, métricas y normas que aseguran la calidad, la seguridad y la usabilidad de los datos durante todo su ciclo de vida, desde la adquisición hasta su eliminación.

477.DATA QUALITY

Proceso y conjunto de estándares para asegurar que los datos sean exactos, completos, consistentes, válidos, únicos y actuales, para que cumplan su propósito y soporten decisiones fiables.

478.DATA CLEANSING

Proceso informático para identificar y corregir o eliminar datos incorrectos, incompletos, duplicados o irrelevantes de un conjunto de datos.

479.DATA PROFILING

Proceso que analiza un conjunto de datos para entender su estructura, contenido y calidad, identificando problemas como valores nulos, duplicados o formatos incorrectos.

480.DATA LINEAGE

Es el registro y la visualización del ciclo de vida completo de los datos, desde su origen hasta su destino final, detallando cada transformación, movimiento y proceso que experimentan los datos.

481.DATA CATALOG

Herramienta que organiza y documenta metadatos sobre los activos de datos de una organización, facilitando su descubrimiento, comprensión y uso adecuado por parte de los usuarios.

482.MDM MASTER DATA MANAGEMENT

Conjunto de procesos, tecnologías y políticas para gestionar y centralizar los datos críticos de una organización (como clientes, productos o ubicaciones) en un único registro maestro.

483.DATA STEWARDSHIP

Práctica de gestionar y supervisar los activos de datos de una organización para asegurar su calidad, accesibilidad, seguridad e integridad.

484.DASHBOARD

Interfaz gráfica de usuario (GUI) que organiza y muestra información clave de forma visual e intuitiva, como métricas y KPIs, para facilitar el monitoreo y la toma de decisiones.

485.KPI (KEY PERFORMANCE INDICATOR)

Métrica cuantitativa que se usa para medir el progreso de un proyecto, un departamento o una empresa hacia el logro de sus objetivos.

486.MÉTRICAS

Mediciones cuantificables de las características de un software, sistema o proceso que ayudan a evaluar, controlar y mejorar la calidad, el rendimiento y el valor de los proyectos de IT.

487.ANÁLISIS DESCRIPTIVO

Proceso de usar datos históricos y actuales para resumir, interpretar y presentar patrones, tendencias y características de un conjunto de datos, con el fin de comprender qué ha sucedido.

488.ANÁLISIS PREDICTIVO

Disciplina de la ciencia de datos que utiliza datos históricos y actuales junto con técnicas de aprendizaje automático y modelos estadísticos para predecir resultados futuros.

489.ANÁLISIS PRESCRIPTIVO

Tipo más avanzado de análisis de datos que recomienda acciones óptimas a tomar basadas en modelos predictivos, inteligencia artificial y algoritmos de optimización.

490.SERIES TEMPORALES

Colección de puntos de datos recopilados secuencialmente a lo largo del tiempo, usualmente en intervalos regulares, para rastrear y comprender su evolución.

491.FORECASTING

Proceso de predecir eventos o resultados futuros analizando patrones y tendencias en datos históricos y actuales, utilizando modelos matemáticos y estadísticos.

492.SEGMENTACIÓN

División de un sistema (como una red, memoria o programa) en partes más pequeñas y manejables, llamadas segmentos.

493.COHORT ANALYSIS

Técnica que agrupa a usuarios o elementos con características comunes en un período específico para rastrear su comportamiento a lo largo del tiempo y descubrir patrones útiles.

494.AB TESTING

Técnica de optimización que compara dos versiones de un elemento digital para determinar cuál produce mejores resultados en una métrica clave.

495.FEATURE STORE

Repositorio centralizado para almacenar, acceder, reutilizar y administrar variables (features) utilizadas en modelos de machine learning, facilitando su uso eficiente y escalable.

496.INGENIERÍA DE SOFTWARE

Disciplina de la ingeniería informática que aplica métodos sistemáticos, disciplinados y cuantificables para el diseño, desarrollo, pruebas, mantenimiento y gestión de sistemas de software.

497.ANÁLISIS DE REQUISITOS

Proceso de recopilar, interpretar y documentar las necesidades y expectativas de los usuarios y partes interesadas para un sistema de software en desarrollo.

498.CASOS DE USO

Describe un conjunto de interacciones entre un actor (usuario o sistema externo) y un sistema para lograr un objetivo específico.

499.HISTORIAS DE USUARIO

Descripción breve e informal de una función de software desde la perspectiva del usuario final, enfocada en el qué y el porqué de la funcionalidad.

500.DIAGRAMAS UML

Representaciones visuales estandarizadas de sistemas de software que permiten especificar, visualizar, construir y documentar la estructura y comportamiento del sistema.

501.DIAGRAMA DE CLASES

Representación gráfica y estática de la estructura de un sistema de software, que utiliza el lenguaje estándar Lenguaje Unificado de Modelado (UML) para mostrar las clases, sus atributos (propiedades), sus métodos (acciones o comportamientos) y las relaciones que existen entre ellas.

502.DIAGRAMA DE SECUENCIAS

Representación gráfica en UML que muestra cómo los objetos de un sistema interactúan entre sí a lo largo del tiempo, ilustrando el orden de los mensajes intercambiados y los procesos que se ejecutan.

503.DIAGRAMA DE CASOS DE USO

Representación gráfica en informática que ilustra la funcionalidad de un sistema desde la perspectiva del usuario, mostrando los actores (usuarios o sistemas externos) y los casos de uso (las funciones que el sistema ofrece).

504.DIAGRAMA DE ACTIVIDAD

Es un tipo de diagrama UML que representa el flujo de actividades o procesos dentro de un sistema, mostrando decisiones, ciclos, acciones y concurrencia. Se usa para modelar procesos de negocio, flujos de trabajo o el comportamiento de métodos.

505.DIAGRAMA DE COMPONENTES

Es un tipo de diagrama UML que modela la arquitectura física de un sistema de software, mostrando cómo se divide en componentes (como archivos fuente, bibliotecas, ejecutables) y las relaciones de dependencia entre ellos.

506.DIAGRAMA DE DESPLIEGUE

Representación visual de la arquitectura de ejecución de un sistema, que muestra los nodos (elementos de hardware como servidores, ordenadores o dispositivos) y cómo los artefactos (componentes de software como ejecutables, bibliotecas o archivos) se asignan a estos nodos físicos.

507.MODELO EN CASCADA

Enfoque lineal y secuencial para el desarrollo de software, donde cada fase del proyecto se completa antes de iniciar la siguiente, fluyendo los resultados como una cascada. Se estructura en etapas como análisis, diseño, implementación, pruebas y mantenimiento.

508.MODELO V

Metodología de desarrollo de software que extiende el modelo en cascada, emparejando cada fase de desarrollo (lado izquierdo de la V) con una fase de prueba correspondiente (lado derecho), enfatizando la verificación y validación del sistema en todo su ciclo de vida.

509.DESARROLLO ÁGIL

Enfoque iterativo e incremental para la ingeniería de software que prioriza la flexibilidad, la colaboración con el cliente y la entrega continua de software funcional.

510.SCRUM

Marco de trabajo ágil para equipos de desarrollo de software que ayuda a gestionar proyectos complejos y adaptativos mediante ciclos de trabajo cortos (llamados Sprints), autoorganización, y entregas incrementales de valor al cliente.

511.KANBAN

Método ágil de gestión de proyectos que utiliza un tablero visual para representar el flujo de trabajo, con el objetivo de maximizar la eficiencia y la transparencia al limitar el trabajo en curso y asegurar una entrega continua de valor.

512.XP (EXTREME PROGRAMMING)

Metodología ágil de desarrollo de software que se enfoca en la calidad del producto, la satisfacción del cliente y la mejora continua del equipo, mediante la implementación de un conjunto de valores y prácticas como TDD, programación en parejas e integración continua.

513.LEAN SOFTWARE DEVELOPMENT

Metodología ágil de desarrollo de software que se enfoca en la entrega de valor al cliente mediante la eliminación de desperdicios, la optimización del tiempo y los recursos, y la mejora continua de los procesos.

514.SPRINT

Ciclo de trabajo de duración fija y corta (generalmente de una a dos semanas) dentro de una metodología ágil, como Scrum, en el que un equipo trabaja para completar una cantidad específica de tareas, culminando en un "incremento" funcional del producto que puede ser entregado a los interesados.

515.PRODUCT BACKLOG

Lista dinámica y priorizada de todo el trabajo pendiente para un producto, que incluye características, mejoras, correcciones de errores, trabajos técnicos y otras actividades necesarias para lograr los objetivos del producto.

516.SPRINT BACKLOG

Es una lista de tareas (historias de usuario, correcciones de errores, etc.) que el equipo de desarrollo se ha comprometido a completar durante un período de tiempo fijo llamado sprint (generalmente de 2 a 4 semanas).

517.DAILY STANDUP

Reunión diaria, corta y de pie que se realiza en los equipos de desarrollo de software que usan metodologías ágiles como Scrum o Kanban. Su objetivo es sincronizar actividades, identificar obstáculos y mejorar la comunicación.

518.PRODUCT OWNER

Persona responsable de maximizar el valor de un producto, actuando como la voz del cliente o usuario final ante el equipo de desarrollo. Define la visión del producto y gestiona el product backlog.

519.SCRUM MASTER

Líder de servicio y facilitador que guía a un equipo de desarrollo ágil a través de la metodología Scrum, eliminando impedimentos y promoviendo buenas prácticas ágiles.

520.EQUIPO DE DESARROLLO

El equipo de desarrollo es el conjunto de personas responsables de diseñar, construir, probar y entregar el software. Incluye desarrolladores, testers, diseñadores, entre otros.

521.VELOCITY

Es una métrica utilizada en metodologías ágiles que representa la cantidad promedio de trabajo completado por el equipo en un sprint. Se expresa generalmente en puntos de historia.

522.BURNDOWN CHART

Es un gráfico que muestra el trabajo restante a lo largo del tiempo en un sprint. Ayuda a visualizar el progreso y detectar desviaciones respecto al plan.

523.BURNUP CHART

Gráfico que muestra el trabajo completado y el alcance total del proyecto. A diferencia del burndown, permite ver si hay cambios en el alcance mientras el proyecto avanza.

524.RETROSPECTIVA

La retrospectiva es una ceremonia ágil que se realiza al final de cada sprint, en la que el equipo reflexiona sobre lo que funcionó, lo que no, y acuerda mejoras para el siguiente sprint.

525.RELEASE PLANNING

Es el proceso de planificar versiones del producto que serán entregadas a los usuarios. Define qué funcionalidades incluir y en qué fechas se liberarán.

526.ROADMAP

Un roadmap es un plan visual a medio o largo plazo que describe los objetivos, entregas clave y la evolución futura del producto o proyecto de software.

527.GESTIÓN DE LA CONFIGURACIÓN

Consiste en controlar y documentar todos los cambios en los artefactos del software para mantener su integridad a lo largo del ciclo de vida.

528.GESTIÓN DE CAMBIOS

La gestión de cambios asegura que cualquier modificación al sistema sea analizada, aprobada y documentada adecuadamente, minimizando riesgos en el proyecto.

529.GESTIÓN DE RIESGO

Implica identificar, evaluar y controlar amenazas que puedan impactar negativamente los objetivos del proyecto.

530.GESTIÓN DE CALIDAD

Se enfoca en asegurar que el software cumpla con los requisitos, estándares y expectativas del cliente mediante prácticas de control y aseguramiento.

531.MÉTRICAS DE SOFTWARE

Indicadores cuantitativos utilizados para evaluar distintos aspectos del desarrollo, como la calidad del código, rendimiento, productividad o defectos.

532.COCOMO (MODELO DE ESTIMACIÓN)

Constructive Cost Model es un modelo utilizado para estimar el esfuerzo, tiempo y costo de desarrollo de software a partir del tamaño del producto y otros factores.

533.PUNTO DE FUNCIÓN

Técnica para medir el tamaño funcional de un sistema a partir de los requerimientos funcionales proporcionados por el usuario.

534.MANTENIMIENTO CORRECTIVO

Implica corregir errores descubiertos después de que el software ha sido implementado. Es esencial para garantizar la estabilidad del sistema.

535.MANTENIMIENTO ADAPTATIVO

Se refiere a los cambios realizados al software para que continúe funcionando en nuevos entornos operativos o tecnológicos.

536.MANTENIMIENTO PERFECTIVO

Tipo de actualización de software que se enfoca en mejorar la funcionalidad, el rendimiento o la usabilidad del sistema.

537.MANTENIMIENTO PREVENTIVO

Enfoque proactivo que consiste en realizar tareas programadas y periódicas de revisión, limpieza, actualización y diagnóstico para anticiparse a los fallos y minimizar los riesgos en sistemas y equipos informáticos.

538.DOCUMENTACIÓN ÁGIL

Práctica de crear y mantener documentación de proyecto utilizando los principios del Manifiesto Ágil, enfocándose en lo esencial, actualizado y útil.

539.PROTOTIPO

Modelo o versión temprana y simplificada de un sistema de software o una aplicación, creado para probar y validar ideas, clarificar requisitos y obtener retroalimentación del cliente antes de desarrollar la versión final.

540.MVP (PRODUCTO MÍNIMO VIABLE)

El tracking de posición permite que el sistema detecte dónde se encuentra el usuario en el espacio real, reflejando movimientos reales en el entorno virtual.

737. CONTROLADORES VR

Los controladores VR son dispositivos que permiten interactuar con el entorno virtual mediante gestos, botones y sensores de movimiento.

738. HAPTICS

739. GUANTES HÁPTICOS

Guantes hápticos son dispositivos que permiten simular sensaciones táctiles en las manos al interactuar con objetos virtuales.

740. TREADMILL VR

Las cintas de correr VR permiten caminar o correr en el entorno virtual mientras se permanece en el mismo sitio en el mundo real.

741. ESPACIOS INMERSIVOS

Los espacios inmersivos son entornos virtuales o mezclados que envuelven al usuario permitiendo una experiencia envolvente total.

742. ESCANEO 3D

El escaneo 3D captura la forma de objetos reales para recrearlos digitalmente en entornos virtuales o aumentados.

743. MODELADO 3D

El modelado 3D es el proceso de crear objetos tridimensionales digitales mediante software especializado.

744. ANIMACIÓN 3D

La animación 3D consiste en dar movimiento a modelos tridimensionales, aplicando transformaciones, esqueletos y deformaciones.

745. MOTORES GRÁFICOS

Los motores gráficos (game engines) procesan gráficos, físicas, audio y entradas para crear entornos virtuales interactivos.

746. UNITY

Es un motor multiplataforma muy usado para desarrollar videojuegos y experiencias VR/AR con alto soporte y comunidad.

747. UNREAL ENGINE

Es un motor gráfico de Epic Games que ofrece renderizado de alta fidelidad y herramientas avanzadas para VR y juegos AAA.

748. GODOT

Es un motor de código abierto para desarrollo de juegos y experiencias 2D/3D, con soporte creciente para VR y AR.

749. WEBXR

Es una API web que permite crear experiencias VR y AR directamente en navegadores compatibles sin necesidad de apps nativas.

750. ARCORE (GOOGLE)

Es la plataforma de Google para desarrollar experiencias de realidad aumentada en dispositivos Android usando cámara y sensores.

751. ARKIT (APPLE)

Es la plataforma de Apple para construir experiencias de RA en dispositivos iOS mediante sensores, cámaras y machine learning.

752. VUFORIA

Es un SDK de RA que permite reconocer imágenes, objetos y superficies para superponer contenido digital en el mundo real.

753. WIKITUDE

Es una plataforma de RA que ofrece herramientas para reconocimiento visual, geolocalización y creación de experiencias AR.

754. LENS STUDIO

Es la herramienta de Snap para crear filtros de RA interactivos y experiencias para Snapchat.

755. SPARK AR

Es la plataforma de Facebook/Meta para crear efectos de RA que funcionan en Instagram y Facebook.

756. FILTROS DE REALIDAD AUMENTADA

Los filtros de RA son efectos visuales aplicados en tiempo real sobre imágenes o video para agregar elementos digitales interactivos.

757. APLICACIONES EDUCATIVAS VR/AR

Las aplicaciones educativas VR/AR usan entornos inmersivos para enseñar de forma interactiva, visualizando conceptos que serían difíciles de mostrar físicamente.

758. SIMULACIONES VR

Las simulaciones VR recrean escenarios realistas en ambientes virtuales para entrenamiento, pruebas o experiencias inmersivas seguras.

759. JUEGOS VR

Los juegos VR permiten interacción total dentro de un mundo virtual, con inmersión visual, auditiva y espacial.

760. MUSEOS VIRTUALES

Los museos virtuales utilizan VR o RA para permitir a los usuarios explorar colecciones culturales de forma inmersiva, sin límite geográfico.

761. TERAPIAS CON VR

Las terapias con VR utilizan entornos virtuales controlados para tratar fobias, ansiedad, estrés postraumático u otras condiciones psicológicas.

762. VISUALIZACIÓN ARQUITECTÓNICA VR

Uso de tecnología de inmersión para crear y explorar entornos arquitectónicos digitales de forma interactiva y realista antes de su construcción física.

763. INDUSTRIA 4.0 CON XR

Sinónimo de fabricación inteligente, representa la materialización de la transformación digital del sector. Ofrece una toma de decisiones en tiempo real, una mayor productividad, flexibilidad y agilidad para revolucionar la manera en que las empresas fabrican, mejoran y distribuyen sus productos.

764. DIGITAL TWINS + XR

Modelo virtual de un objeto físico. Abarca el ciclo de vida del objeto y utiliza los datos en tiempo real enviados por los sensores del objeto para simular el comportamiento y supervisar las operaciones.

765. OCLUSIÓN EN AR

Proceso mediante el cual los objetos virtuales son correctamente ocultados por objetos del mundo real que están físicamente delante de ellos, permitiendo que la experiencia de RA sea más realista e inmersiva.

776. RENDERIZADO EN TIEMPO REAL

Proceso de generar imágenes a partir de modelos 3D en el instante en que son necesarias, permitiendo la interactividad inmediata del usuario con el entorno gráfico, como ocurre en los videojuegos.

777. SHADERS

Es un pequeño programa que se ejecuta en la tarjeta gráfica para calcular y definir la apariencia de un objeto en una pantalla, como el color de un píxel, las texturas, la iluminación y los efectos visuales de un modelo 3D.

778. FOTORREALISMO

Técnica de renderizado 3D que busca crear imágenes digitales que sean casi indistinguibles de una fotografía real, mediante el uso de software especializado para simular el comportamiento físico de la luz, las texturas y los materiales en un entorno virtual.

779. LATENCIA EN VR

Retraso que ocurre entre el momento en que un usuario realiza una acción en una red o aplicación web y el momento en que llega a su destino, que se mide en milisegundos.

780. CINEMÁTICA INVERSA EN VR

Técnica informática que calcula los ángulos de las articulaciones de un modelo (como un personaje o un robot) para que su extremidad o extremo llegue a una posición y orientación específicas deseadas en un espacio virtual.

781. MOTION SICKNESS

Sensación de malestar, náuseas o desorientación provocada por la discrepancia entre el movimiento percibido por los ojos y la información que el cuerpo está "sintiendo", como si fuera un mareo real pero sin movimiento físico.

782. CRIPTOANÁLISIS

La ciencia y el arte de analizar y romper sistemas criptográficos para descubrir sus vulnerabilidades y descifrar mensajes cifrados, incluso sin conocer la clave secreta.

783. ATAQUES DE CANAL LATERAL

Es un tipo de ataque en informática, principalmente contra sistemas criptográficos, que explota información indirecta o "fugas" de un dispositivo físico, como el tiempo que tarda una operación, su consumo de energía, las emisiones electromagnéticas o acústicas, en lugar de intentar romper el algoritmo criptográfico directamente. El objetivo es extraer información sensible, como claves secretas, al analizar estas señales físicas.

784. ATAQUES DE TEMPORIZACIÓN

Ataque de canal lateral en el que el atacante intenta comprometer un criptosistema analizando el tiempo que tarda en ejecutarse un algoritmo criptográfico.

785. ATAQUES DE FUERZA BRUTA DISTRIBUIDA

Es un ciberataque en el que un atacante utiliza una red de miles de computadoras infectadas (una botnet) para probar sistemáticamente todas las combinaciones posibles de contraseñas o claves de cifrado y así obtener acceso no autorizado a un sistema o cuenta, distribuyendo la carga de procesamiento entre múltiples máquinas para acelerar el proceso y dificultar su detección.

786. ATAQUES DE DICCIONARIO DISTRIBUIDO

Es una técnica de piratería informática que combina los ataques de diccionario (usar listas de palabras comunes para adivinar contraseñas) con ataques distribuidos (usar múltiples sistemas para lanzar el ataque simultáneamente), lo que hace que la búsqueda de credenciales sea más rápida y eficiente para los ciberdelincuentes.

787. ATAQUES RAINBOW TABLE

El ataque de tabla arcoiris es un ataque criptográfico que los hackers utilizan para acceder a los sistemas descifrando contraseñas a partir de sus hashes, que actúan como huellas digitales. Una función hash asigna cada contraseña a su cadena de caracteres correspondiente.

788. SEGURIDAD OFENSIVA

Es un enfoque de ciberseguridad que utiliza las mismas tácticas y herramientas que los ciberdelincuentes para simular ataques contra la propia infraestructura de una organización y así descubrir y corregir vulnerabilidades de seguridad antes de que los atacantes reales puedan explotarlas.

789. SEGURIDAD DEFENSIVA

Enfoque proactivo para proteger sistemas, redes y datos mediante la prevención, detección y respuesta a amenazas cibernéticas. Su objetivo es construir sistemas resistentes y mantener barreras de seguridad para salvaguardar los activos de TI contra accesos y ataques no autorizados, asegurando así la confidencialidad, integridad y disponibilidad de la información.

790. INGENIERÍA INVERSA

Proceso de desarmar y analizar un software existente para entender su diseño, funcionamiento interno y estructura, sin tener acceso a la documentación original o al código fuente.

791. MALWARE POLIMÓRFICO

Es un tipo de software malicioso que altera su propio código y firma cada vez que se replica o se propaga, utilizando técnicas de cifrado para cambiar su apariencia mientras mantiene su funcionalidad original. Su principal objetivo es evadir la detección por parte de los antivirus y otros sistemas de seguridad, que buscan patrones de código conocidos para identificar amenazas.

792. MALWARE METAMÓRFICO

Es un tipo de software malicioso que, a diferencia del polimórfico, puede reescribir completamente su propio código en cada infección sin usar claves de cifrado, cambiando su estructura interna y apariencia, pero manteniendo la misma funcionalidad.

793. EXPLOIT

Un exploit es un fragmento de código o técnica que aprovecha una vulnerabilidad en un sistema para ejecutar acciones no autorizadas o lograr un comportamiento inesperado.

794. EXPLOIT KIT

Es un conjunto automatizado de exploits que detecta vulnerabilidades en sistemas y despliega payloads maliciosos si el sistema es vulnerable.

795. PAYLOAD

Es la parte del exploit que toma acción maliciosa (instalar malware, robar datos, abrir puerta trasera) tras explotar una vulnerabilidad.

796. BUFFER OVERFLOW

El buffer overflow ocurre cuando se escriben más datos de los que un búfer puede contener, sobrescribiendo memoria adyacente y pudiendo causar ejecución de código malicioso.

797. HEAP OVERFLOW

Es un desbordamiento en la región de memoria dinámica (heap), donde datos extras sobrescriben estructuras de memoria adyacentes en el heap, pudiendo corromper datos o permitir exploits.

798. INTEGER OVERFLOW

El integer overflow sucede cuando una operación aritmética produce un valor que excede el rango del tipo de dato entero, provocando “wrap around” o resultados incorrectos.

799. USE-AFTER-FREE

Es una vulnerabilidad de memoria donde un programa sigue usando un objeto ya liberado (free), permitiendo que un atacante manipule esa memoria para ataques.

800. DESERIALIZACIÓN INSEGURA

La deserialización insegura ocurre cuando una aplicación acepta datos serializados de fuentes no confiables y los convierte en objetos sin validación, lo que permite inyección de código o manipulación maliciosa.

801. RACE CONDITION EXPLOIT

Un exploit de condición de carrera (race condition) aprovecha el momento en que dos procesos o threads acceden o modifican el mismo recurso simultáneamente, provocando resultados inconsistentes o vulnerables.

802. ESCALADA DE PRIVILEGIOS

La escalada de privilegios es cuando un atacante consigue elevar sus permisos desde un nivel bajo (usuario normal) a uno elevado (administrador/root) en un sistema comprometido.

803. PERSISTENCIA (ATAQUES)

Persistencia en ataques significa mecanismos que permiten al atacante mantener acceso continuo al sistema comprometido, incluso tras reinicios o actualizaciones.

804. EXFILTRACIÓN DE DATOS

La exfiltración de datos es el acto de extraer datos confidenciales de un sistema comprometido hacia un destino controlado por el atacante.

806. COMMAND & CONTROL (C2)

Command & Control (C2) es el canal mediante el cual un atacante controla remotamente sistemas comprometidos (bots), enviando comandos y recibiendo información.

807. APT (AMENAZA PERSISTENTE AVANZADA)

Es un ataque prolongado, dirigido y sofisticado, diseñado para infiltrarse y permanecer en redes estratégicas con el fin de espionaje o sabotaje.

808. SUPPLY CHAIN ATTACK

Un ataque a la cadena de suministro compromete proveedores u otra etapa del ecosistema (hardware, software, servicios) para introducir amenazas que luego afectan al producto final.

809. WETERING HOLE ATTACK

Un ataque de watering hole consiste en comprometer sitios web frecuentados por un grupo objetivo para inyectar malware y luego infectar a visitantes específicos.

810. CROSS-SITE SCRIPTING ALMACENADO

El XSS almacenado permite inyectar scripts maliciosos que se guardan en el servidor (por ejemplo en base de datos), y se ejecutan en los navegadores de los usuarios al cargar la página.

811. XSS REFLEJADO

Reflejado es una vulnerabilidad donde el script malicioso se envía con la solicitud (query, URL) y se refleja inmediatamente en la respuesta, ejecutándose en el navegador del usuario.

812. XSS BASADO EN DOM

El XSS basado en DOM ocurre cuando la manipulación del DOM en el cliente (JavaScript) permite ejecutar scripts maliciosos sin intervención del servidor.

813.SOCIEDAD 5.0

describe una sociedad superinteligente donde la tecnología avanzada (IA, IoT, Big Data) se integra con el espacio físico y el ciberespacio para resolver desafíos sociales, ambientales y económicos, priorizando el bienestar humano y la calidad de vida

814. XML EXTERNAL ENTITY (XXE)

XXE es una vulnerabilidad que permite que un documento XML incluya entidades externas maliciosas, resultando en lectura de archivos locales o solicitudes externas arbitrarias.

815.PATH TRAVERSAL

Path Traversal permite al atacante acceder a archivos o directorios fuera del ámbito permitido usando rutas relativas (como “../”), exponiendo archivos sensibles.

816. LDAP INJECTION

LDAP Injection es similar a SQL Injection pero en el protocolo LDAP, permitiendo manipular consultas LDAP para evadir autenticación o modificar datos en un directorio.

817. DIRECTORY LISTING

Es una configuración donde el servidor muestra listado de todos los archivos en un directorio web, lo que puede revelar ficheros sensibles al atacante.

818. SSRF (SERVER SIDE REQUEST FORGERY)

Permite que un atacante induzca al servidor a hacer solicitudes HTTP/HTTPS arbitrarias desde su lado, potencialmente accediendo a recursos internos o locales.

819. CLICKJACKING

El clickjacking consiste en engañar al usuario para que haga clic en algo distinto de lo que él cree, mediante capas invisibles o disimuladas, ejecutando acciones no deseadas.

820. ATAQUE DE RELLENO DE CREDENCIALES

El credential stuffing es un ataque donde el atacante usa combinaciones de usuario/contraseña previamente comprometidas para intentar acceder masivamente a cuentas en otro servicio.

821. CREDENTIAL STUFFING

Es otro nombre para el ataque de relleno de credenciales, donde un atacante prueba credenciales filtradas de otros servicios para acceder a cuentas válidas.

822. KEYLOGGER

Un keylogger es un software o hardware que registra las pulsaciones de teclas del usuario, capturando contraseñas, datos y otras entradas sensibles.

823. SKIMMING DIGITAL

Es la captura de datos de tarjetas (por ejemplo tarjetas de crédito) mediante dispositivos o scripts maliciosos durante transacciones electrónicas.

824. MAGECART

Es un tipo de ataque que inyecta código malicioso (por ejemplo scripts) en formularios de pago de e‑commerce para robar datos de tarjetas en tiempo real.

825. PHISHING DIRIGIDO (SPEAR PHISHING)

El spear phishing es una forma de phishing dirigida, donde el atacante recopila información específica sobre la víctima para hacer un ataque más creíble y personalizado.

826. WHALING

Es un tipo de phishing que apunta a “presas grandes”, como ejecutivos o personas con acceso a datos críticos, usando ataques muy dirigidos y sofisticados.

827. SMISHING

El smishing es phishing mediante SMS o mensajes de texto, engañando a la víctima para que revele información o instale malware.

828. VISHING

El vishing es phishing por voz (telefonía), donde el atacante llama simulando una entidad confiable para extraer datos sensibles o credenciales.

829. ATAQUES HÍBRIDOS

Los ataques híbridos combinan múltiples técnicas (por ejemplo phishing + malware + ingeniería social) para maximizar la efectividad y evitar mecanismos de defensa.

830. SEGURIDAD EN ENTORNOS OT

Seguridad en entornos OT (Operational Technology) se refiere a proteger sistemas de control industrial (SCADA, ICS) y equipos críticos de infraestructura física contra ciberataques.

831. PENTESTING AVANZADO

Pentesting avanzado implica pruebas de penetración profundas y sofisticadas, simulando ataques reales para descubrir vulnerabilidades ocultas en sistemas complejos.

832. RED TEAMING AVANZADO

Red teaming avanzado es una simulación completa de adversario, donde un equipo actúa como el atacante para evaluar la efectividad de defensas, procedimientos y detección.

833. ARQUITECTURA DE COMPUTADORAS

La arquitectura de computadoras es el diseño funcional y estructural de un sistema de cómputo: cómo interactúan el procesador, memoria, buses, etc.

834. INSTRUCCIÓN

Una instrucción es una operación básica que la unidad de CPU puede ejecutar (por ejemplo suma, lectura de memoria). Forma parte del conjunto de instrucciones del procesador.

835. CONJUNTO DE INSTRUCCIONES (ISA)

ISA (Instruction Set Architecture) es el conjunto de instrucciones y operaciones que un procesador soporta y que el software puede invocar.

836. RISC (REDUCED INSTRUCTION SET COMPUTER)

RISC es una arquitectura de procesador con un conjunto reducido de instrucciones simples que permite una ejecución rápida y eficiente.

837. CISC (COMPLEX INSTRUCTION SET COMPUTER)

CISC es una arquitectura con instrucciones más complejas y numerosas, cada una potencialmente ejecutando múltiples operaciones de bajo nivel en un solo comando.

838. ARM

Es una familia de arquitecturas RISC ampliamente usada en dispositivos móviles, embebidos y sistemas con eficiencia energética.

839. X86

Es una arquitectura de conjunto de instrucciones (ISA) desarrollada por Intel, usada en PCs y servidores.

840. X64

(O x86-64) es la extensión de 64‑bits de la arquitectura x86, permitiendo mayor espacio de direcciones y operaciones con datos grandes.

841. MIPS

Es una arquitectura RISC usada en embebidos, routers y consolas, conocida por su simplicidad y eficiencia en pipelines.

842. SPARC

SPARC es una arquitectura RISC desarrollada por Sun Microsystems, usada en servidores empresariales y sistemas Unix.

843. PIPELINE

Es una técnica de CPU donde las diferentes etapas de ejecución (fetch, decode, execute, etc.) se superponen para procesar múltiples instrucciones en paralelo.

844. SUPERESCALAR

Una arquitectura superescalar permite que múltiples instrucciones sean ejecutadas simultáneamente en una CPU, usando múltiples unidades de ejecución.

845. MULTINÚCLEO

Una CPU multinúcleo integra varios núcleos de procesamiento independientes en el mismo chip, permitiendo paralelismo y mejor rendimiento en tareas concurrentes.

846. HYPER-THREADING

Es la capacidad de un núcleo de CPU para manejar múltiples hilos de ejecución simultáneos, simulando múltiples “núcleos lógicos”.

847. SIMD (SINGLE INSTRUCTION MULTIPLE DATA)

Es una técnica de paralelismo de datos donde una sola instrucción opera simultáneamente sobre múltiples datos (por ejemplo vectores) en una CPU.

848. MIMD (MULTIPLE INSTRUCTION MULTIPLE DATA)

MIMD es un modelo de paralelismo donde múltiples instrucciones operan sobre múltiples datos simultáneos, típico en sistemas de multiprocesamiento.

849. HARVARD CACHE

La arquitectura Harvard separa memoria e instrucciones y datos, permitiendo accesos simultáneos a instrucciones y datos sin conflictos de caché.

850. L1 CACHE

La caché L1 es la memoria caché más rápida y cercana al núcleo de la CPU, utilizada para almacenar datos e instrucciones de acceso muy frecuente.

851. L2 CACHE

La caché L2 es una memoria caché intermedia entre L1 y la memoria principal, con capacidad mayor que L1 y menor velocidad.

852. L3 CACHE

La caché L3 es una caché compartida entre múltiples núcleos de CPU, con mayor capacidad pero menor velocidad comparada con L1 y L2.

853. UNIDAD DE CONTROL

La unidad de control es el componente de la CPU que interpreta instrucciones del programa y coordina otras partes de la CPU para ejecutar esas instrucciones.

854. UNIDAD ARITMÉTICO - LÓGICA (ALU)

La ALU (Arithmetic Logic Unit) es la parte de la CPU que realiza operaciones aritméticas (suma, resta) y lógicas (AND, OR, comparaciones).

855. UNIDAD DE PUNTO FLOTANTE (FPU)

La FPU (Floating Point Unit) es la parte de la CPU que maneja cálculos en punto flotante (decimales, operaciones científicas, etc.).

856. COPROCESADOR

Un coprocesador es un procesador adicional especializado (por ejemplo para matemáticas o gráficos) que trabaja en conjunto con la CPU principal.

857. CICLO DE INSTRUCCIÓN

El ciclo de instrucción es el proceso que realiza la CPU para ejecutar una instrucción: fetch, decode, execute, write back (y otras etapas auxiliares).

858. FETCH

Es la etapa del ciclo de instrucción donde la CPU lee la siguiente instrucción de memoria para ejecutarla.

859. DECODE

Decode (decodificar) es la etapa donde la CPU interpreta el código de la instrucción para determinar qué operación se debe realizar y qué operandos usar.

860.EXECUTE

Execute (ejecutar) es la fase en la que la CPU realiza la operación definida por la instrucción (aritmética, lógica, memoria, etc.).

861. WRITE BACK

Write Back (escribir de vuelta) es la etapa donde el resultado de la instrucción se almacena en el registro o memoria correspondiente.

862. PREFETCH

Prefetch es la técnica de anticipar y cargar instrucciones o datos en caché antes de que sean necesarios, para mejorar el rendimiento.

863. BRANCH PREDICTION

Es una técnica para que la CPU “adivine” la dirección de una rama condicional (if, loops) para mantener su pipeline lleno y optimizar ejecución.

864. SPECULATIVE EXECUTION

Es cuando la CPU ejecuta instrucciones “adelantadas” basadas en predicción para ganar tiempo, aunque algunas podrían descartarse si la predicción fue incorrecta.

865. OUT-OF-ORDER EXECUTION

La ejecución fuera de orden permite que la CPU ejecute instrucciones no necesariamente en el orden que aparecen, sino cuando sus operandos estén listos, para optimizar el uso de unidades funcionales.

866. REGISTER RENAMING

El register renaming es una técnica que evita dependencias falsas entre instrucciones al asignar nombres lógicos a registros físicos para paralelismo y evitar conflictos.

867. BUS DE SISTEMAS

El bus de sistema es el canal de comunicación que conecta la CPU, memoria y otros componentes del sistema para transferir datos e instrucciones.

868. MEMORIA VIRTUAL

La memoria virtual es una técnica que permite que los programas utilicen más memoria de la físicamente disponible, usando espacio en disco como extensión de la memoria RAM.

869. PAGINACIÓN

La paginación es un esquema de memoria en el que el espacio de direcciones de un proceso se divide en páginas de igual tamaño, mapeadas a marcos en memoria física.

870. SEGMENTACIÓN DE MEMORIA

La segmentación divide la memoria lógica en segmentos de longitud variable (por ejemplo código, datos, pila), cada uno con su base y longitud.

871. MMU (MEMORY MANAGEMENT UNIT)

Es un componente hardware que traduce las direcciones virtuales en direcciones físicas, manejando paginación, protecciones y fallos de página.

872. TLB (TRANSLATION LOOKASIDE BUFFER)

El TLB es una caché de traducciones recientes de direcciones virtuales a físicas, utilizada para acelerar la traducción de direcciones en la MMU.

873. INSTRUCCIÓN PRIVILEGIADA

Una instrucción privilegiada es aquella que sólo puede ejecutarse cuando la CPU está en modo kernel o supervisor; no puede ejecutarse en modo usuario para evitar abusos.

874. DMA (DIRECT MEMORY ACCESS)

Es una técnica que permite a dispositivos periféricos transferir datos directamente hacia/desde memoria sin intervención de la CPU, liberando su carga.

875. INTERRUPCIÓN HARDWARE

Una interrupción de hardware es una señal generada por dispositivos (ej. clock, I/O) para notificar a la CPU que requieren atención inmediata.

876. INTERRUPCIÓN SOFTWARE

Es generada por instrucciones especiales (trap, syscall) dentro del procesador para cambiar el flujo de ejecución y llamar al sistema operativo.

877. ARQUITECTURA NUMA

Non-Uniform Memory Access es una arquitectura donde el tiempo de acceso a memoria depende de la ubicación del procesador y de la memoria—algunos accesos son más rápidos que otros.

878. ARQUITECTURA SMP

(Symmetric Multiprocessing) es un diseño en el que múltiples procesadores comparten la misma memoria física y tienen los mismos privilegios de acceso.

879. ARQUITECTURA DISTRIBUIDA

Una arquitectura distribuida está formada por múltiples nodos independientes que cooperan a través de comunicaciones de red para lograr un objetivo común.

880. COMPUTACIÓN PARALELA

La computación paralela consiste en ejecutar múltiples operaciones simultáneamente para resolver un problema más rápido dividiéndolo en tareas concurrentes.

881. COMPUTACIÓN HETEROGÉNEA

En computación heterogénea se combinan distintos tipos de procesadores (CPU, GPU, FPGA) para aprovechar sus fortalezas específicas en un sistema conjunto.

882. ARQUITECTURA CUÁNTICA

La arquitectura cuántica se basa en qubits y fenómenos de superposición y entrelazamiento para realizar cómputos que pueden superar paradigmas clásicos en ciertas tareas.

883. RED INALÁMBRICA

Una red inalámbrica permite la comunicación entre dispositivos sin cables, utilizando ondas de radio, infrarrojo u otras tecnologías electromagnéticas.

884. WI-FI

Wi-Fi es un estándar de red inalámbrica para interconectar dispositivos en redes locales sin cables usando protocolos IEEE 802.11.

885. WI-FI 5 (802.11AC)

Wi-Fi5, o 802.11ac, es una versión del estándar Wi-Fi que opera en la banda de 5 GHz y ofrece mayores velocidades y eficiencia que versiones anteriores.

886. WI-FI 6 (802.11AX)

Wi-Fi6, o 802.11ax, mejora el rendimiento en entornos densos, con mayor eficiencia espectral, menor latencia y capacidad para más dispositivos.

887. WI-FI 7

Wi-Fi7 es la próxima generación del estándar Wi-Fi, con mejoras en velocidad, eficiencia y menor latencia, usando técnicas avanzadas de modulación y canalización.

888. BLUETOOTH LOW ENERGY (BLE)

BLE es una variante de Bluetooth diseñada para comunicaciones de bajo consumo energético, ideal para dispositivos IoT y sensores.

889. NFC (NEAR FIELD COMMUNICATION)

NFC es una tecnología de comunicación inalámbrica de corto alcance (unos pocos centímetros) usada en pagos móviles, tarjetas y emparejamientos rápidos.

890. RFID

Radio-Frequency Identification es una tecnología para identificar objetos o personas mediante etiquetas que responden a señales de radio.

891. 2G

2G es la segunda generación de redes móviles digitales, introduciendo servicios como llamadas y SMS con conmutación de circuitos.

892. 3G

Es la tercera generación de redes móviles, que permitió servicios de datos móviles más rápidos (navegación, video) junto con voz.

893. 4G LTE

Es una tecnología de cuarta generación con acceso de datos de alta velocidad, menor latencia y mejor eficiencia espectral.

894. 5G

5G, ofreciendo velocidades muy altas, latencia ultrabaja y conectividad masiva para IoT, es la quinta generación de redes móviles.

895. 6G

Es la futura generación de redes móviles esperada para avanzar en velocidad, integración con inteligencia artificial y ultra conectividad.

896. eSIM

Es una SIM embebida, programable remotamente, que permite cambiar de operador sin tener que reemplazar la tarjeta física.

897. IMSI

(International Mobile Subscriber Identity) es un identificador único asignado al usuario en redes móviles, almacenado en la SIM/eSIM.

898. APN (ACCESS POINT NAME)

Es el nombre de punto de acceso usado por el dispositivo móvil para conectarse a la red de datos del operador y alcanzar Internet.

899. CELL TOWER

Una cell tower es una estación base que proporciona cobertura de red celular a dispositivos en su área mediante antenas y transmisores.

900. ROAMING

El roaming permite que un dispositivo móvil use redes de operadores distintos a su operador base cuando está fuera de su área de cobertura.

901. HAND-OFF

Hand-Off (o handover) es el proceso por el cual la conexión de un dispositivo se transfiere de una celda (torre) a otra sin interrumpir la comunicación.

902. SATÉLITE GEOESTACIONARIO

Un satélite geoestacionario órbita a la misma velocidad que la rotación de la Tierra, permaneciendo fijo sobre un punto del ecuador.

903. SATÉLITE LEO

Los satélites LEO (Low Earth Orbit) orbitan cerca de la Tierra, proporcionando menor latencia y mejor cobertura para servicios como internet satelital.

904. STARLINK

Es una constelación de satélites LEO de SpaceX diseñada para proporcionar acceso a Internet de alta velocidad a nivel global.

905. TELEFONÍA VOIP

(Voice over IP) es una tecnología que permite la transmisión de voz por redes IP (Internet), convirtiendo voz en paquetes de datos.

906. SIP (SESSION INITIATION PROTOCOL)

Es un protocolo de señalización para establecer, modificar y terminar sesiones de comunicación multimedia (voz, video) en redes IP.

907. RTP (REAL-TIME TRANSPORT PROTOCOL)

Es un protocolo para transmitir datos en tiempo real (voz, video) sobre IP, asegurando sincronización y secuenciación.

908. WEBRTC

Es un conjunto de estándares que permite comunicación en tiempo real (audio, video, datos) directamente entre navegadores sin necesidad de plugins.

909. IMS (IP MULTIMEDIA SUBSYSTEM)

Es una arquitectura de red para servicios multimedia sobre IP (voz, video, mensajería) integrada con redes celulares y redes fijas.

910. MPLS INÁLAMBRICO

MPLS inalámbrico combina la tecnología MPLS (Multi-Protocol Label Switching) con enlaces inalámbricos para mejorar la enrutación y calidad de servicio en redes móviles o fijas inalámbricas.

911. HOTSPOT

Un hotspot es un punto de acceso Wi-Fi público o privado que permite a los dispositivos conectarse a Internet inalámbricamente en esa zona.

912. ANTENAS MIMO

MIMO (Multiple Input, Multiple Output) emplea múltiples antenas en transmisor y receptor para mejorar la capacidad y fiabilidad del enlace inalámbrico.

913. BEAMFORMING

Beamforming es una técnica de antenas donde las señales se focalizan hacia el receptor en lugar de irradiar en todas las direcciones, mejorando alcance y calidad.

914. OFDM (ORTHOGONAL FREQUENCY DIVISION MULTIPLEXING)

Es una técnica de modulación que divide un canal en múltiples subportadoras ortogonales para transmitir datos simultáneamente, reduciendo interferencias.

915. DSSS (DIRECT SEQUENCE SPREAD SPECTRUM)

Es una técnica de espectro ensanchado donde la señal original se multiplica por una secuencia de código de alta velocidad, distribuyendo su energía y haciendo más difícil interferir.

916. FHSS (FREQUENCY HOPPING SPREAD SPECTRUM)

Es otra técnica de espectro ensanchado donde la señal salta entre múltiples frecuencias de forma pseudoaleatoria, dificultando interferencias o interceptación.

917. BANDA ISM

Las bandas ISM (Industrial, Scientific and Medical) son rangos de frecuencias reservadas para uso no comercial, empleadas en aplicaciones como Wi-Fi, Bluetooth, RFID sin licencia.

918. BANDA LICENCIADA

Una banda licenciada es un rango de frecuencia que requiere autorización y pago por parte de autoridades reguladoras para operar legalmente, usada por operadores móviles y servicios críticos.

919. BANDA ANCHA MÓVIL

La banda ancha móvil se refiere a servicios de internet de alta velocidad accesibles sobre redes móviles (3G, 4G, 5G) que permiten transmisión de datos intensiva.

920. BANDA ESTRECHA

La banda estrecha usa un ancho de banda pequeño para comunicaciones de baja tasa de datos, ideal para IoT, sensores y transmisores de bajo consumo.

921. RADIO DEFINIDA POR SOFTWARE (SDR)

Es un sistema de radio donde muchas funciones (modulación, demodulación, filtrado) se implementan en software en lugar de hardware fijo, permitiendo flexibilidad.

922. LTE-M

LTE-M es una tecnología de comunicación celular para IoT que ofrece cobertura amplia, bajo consumo y costo reducido en conexiones de sensores y dispositivos.

923. NB-IOT

NB-IoT es una tecnología de banda estrecha optimizada para dispositivos IoT, ofreciendo cobertura en interiores, bajo consumo y soporte masivo de conexiones.

924. WIMAX

Es una tecnología de banda ancha inalámbrica (IEEE 802.16) destinada a ofrecer acceso a internet fijo o móvil en áreas extensas.

925. ZIGBEE PRO

Zigbee Pro es una especificación del estándar Zigbee optimizada para redes de sensores de largo alcance, baja latencia y baja potencia en ambientes IoT.

926. Z-WAVE PLUS

Es una versión mejorada del protocolo Z-Wave para domótica, con mayor alcance, seguridad y eficiencia energética.

927. SEGURIDAD WI-FI (WPA2, WPA3)

Son estándares de seguridad para redes Wi-Fi que proporcionan cifrado y autenticación robusta para proteger la comunicación inalámbrica.

928. CAPTIVE PORTAL

Un captive portal es una página web interceptada por un punto de acceso inalámbrico que solicita autenticación (usuario/clave) antes de permitir acceso a Internet.

929. WARDRIVING

Es la práctica de recorrer áreas para detectar redes inalámbricas abiertas o vulnerables usando herramientas como laptops, smartphones y escáneres Wi-Fi.

930. ANTENAS DIRECCIONALES

Las antenas direccionales enfocan la transmisión en una dirección concreta, en lugar de irradiar en todas direcciones, incrementando alcance y ganancia en ese rumbo.

931. ROBÓTICA

La robótica es la rama de ingeniería que estudia el diseño, construcción, operación y uso de robots para realizar tareas automáticas o asistidas.

932. ROBOT INDUSTRIAL

Un robot industrial es un manipulador automático, programable y reconfigurable, empleado en entornos industriales para tareas repetitivas como soldadura, montaje o pintura.

933. ROBOT COLABORATIVO (COBOT)

Un cobot es un robot diseñado para interactuar y trabajar de forma segura junto a humanos, colaborando en tareas sin barreras de seguridad rígidas.

934. ROBOT AUTÓNOMO

Un robot autónomo opera sin intervención humana directa, percibiendo el entorno y tomando decisiones por sí mismo para moverse o actuar.

935. ROBOT MÓVIL

Un robot móvil puede desplazarse en su entorno usando ruedas, orugas, patas o cualquier mecanismo de locomoción, navegando sin cableado fijo.

936. AMR (AUTONOMOUS MOBILE ROBOT)

Es un robot móvil autónomo diseñado para navegar dinámicamente en entornos flexibles (almacenes, fábricas), evitando obstáculos sin infraestructura fija.

937. AGV (AUTOMATED GUIDED VEHICLE)

Es un vehículo guiado automáticamente (por rieles, banda magnética o láser) para transportar cargas en entornos controlados como almacenes o plantas.

938. BRAZO ROBÓTICO

Un brazo robótico es un tipo de robot manipulador con articulaciones y eslabones, usado para mover objetos con precisión en múltiples grados de libertad.

939. MANIPULADOR

El manipulador es la parte física de un robot que interactúa con el entorno (brazo, garra), ejecutando movimientos y operaciones sobre objetos.

940. ACTUADOR ROBÓTICO

Un actuador robótico convierte señales de control en movimiento físico (rotación, traslación) para que el robot ejecute acciones.

941. SERVOMOTOR

Un servomotor es un tipo de actuador con control de posición, velocidad y torque, muy usado en robótica para movimientos precisos.

942. MOTOR PASO A PASO

Un motor paso a paso se mueve en pasos discretos, permitiendo control exacto de posición sin necesidad de sensores de retroalimentación en muchas aplicaciones.

943. MOTOR DC

Un motor DC (corriente continua) convierte energía eléctrica en movimiento giratorio continuo, utilizado ampliamente en robótica y mecanismos.

944. SENSOR DE PROXIMIDAD

Un sensor de proximidad detecta la presencia de objetos cercanos sin contacto físico, usando tecnologías como ultrasonido, infrarrojo o campos electromagnéticos.

945. SENSOR DE DIVISIÓN

Un sensor de disipación (o sensor de división) detecta eventos rápidos o rupturas (por ejemplo, rotura de haz de luz) para activar acciones inmediatas.

946. SENSOR LIDAR

Un sensor LiDAR mide distancias y genera mapas 3D del entorno mediante pulsos láser, siendo muy usado en vehículos autónomos y robótica.

947. SENSOR ULTRASÓNICO

El sensor ultrasónico emite ondas sonoras de alta frecuencia y mide el tiempo de eco para calcular distancias a objetos.

948. SENSOR INFRARROJO

Un sensor infrarrojo detecta objetos o movimiento mediante emisión/detección de radiación infrarroja, común en control remoto, proximidad o línea.

949. SENSOR DE FUERZA

Un sensor de fuerza mide la magnitud de una fuerza aplicada (por ejemplo presión, peso) y lo convierte en señal eléctrica.

950. CINEMÁTICA DIRECTA

La cinemática directa calcula la posición final del efector de un robot (por ejemplo brazo) a partir de los ángulos de las articulaciones.

951. CINEMÁTICA INVERSA

La cinemática inversa calcula los ángulos de las articulaciones necesarios para que el efector alcance una posición deseada en el espacio.

952. DINÁMICA DE ROBOTS

La dinámica estudia las fuerzas y torques que producen movimientos en robots, relacionando masa, aceleración, velocidades y cargas.

953. CONTROL PID

El control PID (Proporcional-Integral-Derivativo) ajusta una señal de control para minimizar el error entre una variable medida y su referencia usando tres términos.

954. CONTROL ADAPTATIVO

El control adaptativo ajusta sus parámetros dinámicamente en respuesta a cambios en el sistema o variaciones del entorno para mantener el desempeño.

955. CONTROL ROBUSTO

El control robusto busca diseñar sistemas que mantengan estabilidad y desempeño aceptable frente a incertidumbres y perturbaciones del modelo.

956. SLAM (SIMULTANEOUS LOCALIZATION AND MAPPING)

Es el proceso por el cual un robot construye un mapa del entorno mientras estima su posición dentro de ese mapa simultáneamente.

957. PLANIFICACIÓN DE TRAYECTORIAS

La planificación de trayectorias consiste en calcular rutas o caminos que el robot debe seguir para moverse de un punto A a B evitando obstáculos.

958. NAVEGACIÓN AUTÓNOMA

La navegación autónoma permite que un robot se desplace por su entorno sin intervención humana, tomando decisiones sobre ruta y movimiento.

959. ROBOT HUMANOIDE

Un robot humanoide está diseñado con forma humana (cabeza, torso, brazos, piernas) para interactuar con entornos desarrollados para humanos.

960. BIPEDESTACIÓN

La bipedestación es la capacidad de mantenerse en pie sobre dos piernas, necesaria para robots humanoides o exoesqueletos que imitan la postura humana.

961. EXOESQUELETO

Un exoesqueleto es una estructura robótica portátil que apoya o mejora las capacidades físicas humanas (fuerza, movilidad) mediante actuadores mecánicos.

962. DRON

(UAV, vehículo aéreo no tripulado) es un dispositivo volador autónomo o remotamente pilotado usado para vigilancia, entrega, investigación, etc.

963. DRON CUADRICÓPTERO

Un dron cuadricóptero es un tipo de dron con cuatro rotores, popular por su estabilidad, maniobrabilidad y facilidad de control.

964. DRON HEXACÓPTERO

Un dron hexacóptero tiene seis rotores, ofreciendo mayor capacidad de carga y redundancia frente a fallos en comparación con los cuadricópteros.

965. DRON VTOL

Un dron VTOL (Vertical Take-Off and Landing) es capaz de despegar y aterrizar verticalmente como un helicóptero, y luego volar horizontalmente como un avión.

966. DRON SUBMARINO

Un dron submarino es un vehículo no tripulado diseñado para operar bajo el agua. Se utiliza en exploración marina, investigación científica y vigilancia.

967. SWARM ROBOTICS

Swarm Robotics se refiere a sistemas de múltiples robots que trabajan de manera coordinada como un enjambre, inspirados en comportamientos de insectos sociales.

968. TELEOPERACIÓN

La teleoperación es el control remoto de un robot o máquina desde una distancia, común en entornos peligrosos o inaccesibles para humanos.

969. ROBÓTICA MÉDICA

La robótica médica aplica tecnologías robóticas en el campo de la medicina, mejorando precisión quirúrgica, rehabilitación y asistencia al paciente.

970. CIRUGÍA ROBÓTICA

La cirugía robótica permite a los cirujanos realizar operaciones mediante brazos robóticos con alta precisión y control desde una consola.

971. ROBÓTICA AGRÍCOLA

La robótica agrícola automatiza tareas en el campo como la siembra, cosecha y monitoreo, aumentando la eficiencia y reduciendo el trabajo manual.

972. ROBÓTICA DE EXPLORACIÓN ESPACIAL

Esta rama de la robótica desarrolla sistemas robóticos para operar en ambientes espaciales, incluyendo satélites, sondas y vehículos de exploración.

973. ROVERS ESPACIALES

Los rovers espaciales son vehículos robóticos diseñados para desplazarse sobre la superficie de otros planetas, recolectando datos e imágenes.

974. MARS ROVER PERSEVERANCE

El Perseverance es un rover de la NASA que aterrizó en Marte en el año 2021 para buscar signos de vida pasada y recolectar muestras geológicas.

975. BOSTON DYNAMICS

Boston Dynamics es una empresa líder en robótica conocida por sus robots avanzados como Atlas y Spot, capaces de movimientos ágiles y complejos.

976. ROBOT ATLAS

Atlas es un robot humanoide desarrollado por Boston Dynamics para realizar tareas dinámicas como correr, saltar y manipular objetos.

977. ROBOT SPOT

Spot es un robot cuadrúpedo también de Boston Dynamics diseñado para inspecciones, seguridad y tareas industriales en terrenos difíciles.

978. SOFT ROBOTICS

Soft Robotics se enfoca en el diseño de robots con materiales blandos y flexibles que permiten movimientos seguros e interactivos con humanos.

979. ÉTICA EN ROBÓTICA

La ética en robótica analiza los dilemas morales y sociales que surgen del uso de robots, como su impacto en el empleo, privacidad y autonomía.

980. LEYES DE LA ROBÓTICA (ASIMOV)

Las tres leyes de la robótica, formuladas por Isaac Asimov, establecen principios para asegurar que los robots actúen en beneficio de los humanos.

981. COMPUTACIÓN CUÁNTICA

La computación cuántica utiliza principios de la mecánica cuántica para realizar cálculos mucho más rápido que las computadoras clásicas en ciertos problemas.

982. QUBIT

El qubit es la unidad básica de información en la computación cuántica, que puede estar en múltiples estados simultáneamente gracias a la superposición.

983. SUPERPOSICIÓN CUÁNTICA

La superposición permite a un qubit representar varios estados al mismo tiempo, lo que potencia el paralelismo en cálculos cuánticos.

984. ENTRELAZAMIENTO CUÁNTICO

El entrelazamiento cuántico es un fenómeno donde dos qubits están conectados de tal manera que el estado de uno afecta al otro instantáneamente.

985. PUERTA CUÁNTICA

Las puertas cuánticas son operaciones que alteran el estado de los qubits, similares a las compuertas lógicas en computadoras clásicas pero con mayor complejidad.

986. CIRCUITO CUÁNTICO

Un circuito cuántico es una secuencia de puertas cuánticas aplicadas a qubits para realizar un algoritmo o cálculo específico.

987. ALGORITMO DE SHOR

El algoritmo de Shor permite factorizar números grandes de manera eficiente en computadoras cuánticas, amenazando la criptografía tradicional.

988. ALGORITMO DE GROVER

El algoritmo de Grover mejora la búsqueda en bases de datos no ordenadas, ofreciendo una ventaja cuadrática sobre métodos clásicos.

999. ALGORITMO CUÁNTICO DE DEUTSCH-JOZSA

Este algoritmo cuántico demuestra la ventaja cuántica al resolver ciertos problemas con una sola consulta, frente a múltiples en computación clásica.

1000. TELEPORTACIÓN CUÁNTICA

Es un proceso que permite transferir el estado cuántico de una partícula a otra distante, usando entrelazamiento y medición cuántica.

1001. ERROR CUÁNTICO

Los errores cuánticos son alteraciones en los estados de los qubits, producto de la interferencia o de coherencia en sistemas cuánticos.

1002. CORRECCIÓN DE ERRORES CUÁNTICOS

Métodos desarrollados para detectar y corregir errores en qubits sin colapsar su estado cuántico.

1003. DECOHERENCIA

La decoherencia es la pérdida de información cuántica debido a la interacción con el entorno, lo cual limita los cálculos cuánticos.

1004. MEDICIÓN CUÁNTICA

Proceso mediante el cual el estado de un qubit se colapsa a un valor definido al ser observado.

1005. QAOA (QUANTUM APPROXIMATE OPTIMIZATION ALGORITHM)

Algoritmo cuántico híbrido que combina computación cuántica y clásica para resolver problemas de optimización.

1006. VQE (VARIATIONAL QUANTUM EIGENSOLVER)

Algoritmo que usa circuitos cuánticos parametrizados para encontrar valores propios mínimos de operadores Hamiltonianos, útil en química cuántica.

1007. QUANTUM SUPREMACY

Se refiere al punto en que una computadora cuántica realiza una tarea que sería prácticamente imposible para una clásica.

1008. QUANTUM ADVANTAGE

Ventaja cuántica es cuando un computador cuántico ofrece mejoras prácticas y significativas frente a los clásicos en tareas reales.

1009. QISKIT

Es un framework de código abierto de IBM para desarrollar programas de computación cuántica en Python.

1010. CIRQ

Es una biblioteca de código abierto desarrollada por Google para crear y ejecutar circuitos cuánticos en sus procesadores cuánticos.

1011. BRAKET (AWS)

Amazon Braket es un servicio en la nube de AWS que permite desarrollar, probar y ejecutar algoritmos cuánticos en hardware real o simulado.

1012. AZURE QUANTUM

Es la plataforma de Microsoft que integra herramientas, hardware y simuladores para computación cuántica.

1013. IBM QUANTUM EXPERIENCE

Plataforma en la nube de IBM que permite a los usuarios ejecutar programas en computadoras cuánticas reales.

1014. D-WAVE

D-Wave es una empresa pionera en computación cuántica que desarrolla computadores basados en el modelo de recocido cuántico (annealing).

1015. ANNEALING CUÁNTICO

Técnica utilizada en computación cuántica para resolver problemas de optimización usando energía mínima.

1016. COMPUTACIÓN CUÁNTICA UNIVERSAL

Modelo teórico de computación cuántica capaz de simular cualquier algoritmo, similar a la Turing universal pero cuántica.

1017. POST-CUÁNTICO

Criptografía post-cuántica se refiere a técnicas resistentes a ataques por computadores cuánticos futuros.

1018. CRIPTOGRAFÍA CUÁNTICA

Uso de principios cuánticos para asegurar la transmisión de información, como la distribución de claves seguras.

1019.QKD(QUANTUM KEY DISTRIBUTION)

Método de criptografía cuántica que permite el intercambio seguro de claves gracias al entrelazamiento cuántico.

1020. RED CUÁNTICA

Una red cuántica conecta dispositivos cuánticos para distribuir qubits y entrelazamiento entre nodos de manera segura.

1021.SIMULACION CUANTICA

consiste en usar computadoras clásicas para imitar el comportamiento de sistemas cuánticos, actuando como una alternativa de bajo costo a las computadoras cuánticas reales, que son escasas y caras. Permite investigar y desarrollar algoritmos cuánticos, probar la supremacía cuántica, y simular el funcionamiento de sistemas complejos en áreas como química, medicina y aprendizaje automático.

1022.LENGUAJE Q#

Es un lenguaje de programación desarrollado por Microsoft para la computación cuántica, compatible con Visual Studio y Azure Quantum.

1023.LENGUAJE OPEN QASM

Lenguaje intermedio utilizado por Qiskit para representar instrucciones cuánticas, compatible con múltiples plataformas.

1924.HARDWARE CUANTICO SUPERCONDUCTORES

El hardware cuántico superconductores implementa qubits usando materiales superconductores a temperaturas criogénicas, aprovechando corrientes sin resistencia y estructuras de Josephson.

1025.IONES ATRAPADOS

Los iones atrapados son una tecnología de qubits donde iones individuales se aíslan en campos electromagnéticos y se manipulan con láseres para operaciones cuánticas.

1026.FOTONICA CUANTICA

La fotónica cuántica utiliza fotones (partículas de luz) como qubits o portadores cuánticos, aprovechando su manipulación mediante óptica cuántica.

1027.COMPUTACION CUANTICA HIBRIDA

La computación cuántica híbrida combina recursos cuánticos y clásicos para resolver problemas, delegando partes al modelo óptimo para cada dominio.

1028.BENCHMARK CUANTICO

Un benchmark cuántico es una prueba estándar para comparar el rendimiento de diferentes sistemas cuánticos, midiendo métricas como fidelidad, velocidad o volumen cuántico.

1029.NISQ (NOISY INTERMEDIATE-SCALE QUANTUM)

1347.PURPLE TEAM

Equipo híbrido que combina las capacidades de ataque del Red Team y las defensas del Blue Team para mejorar la seguridad de una organización de forma colaborativa y continua.

1348.THREAT HUNTING

Proceso de ciberseguridad proactivo donde expertos en seguridad buscan activamente amenazas y actividades maliciosas que hayan evadido las defensas de seguridad automatizadas, como firewalls o antivirus, en una red o sistema informático.

1349.THREAT INTELLIGENCE

Proceso de recopilar, analizar y difundir información sobre ciberamenazas para que las organizaciones puedan anticipar y defenderse mejor de ataques presentes y futuros.

1350.ZERO-DAY EXPLOIT

Es un ataque informático que aprovecha una vulnerabilidad desconocida en un software o hardware, sobre la cual el proveedor o desarrollador no tiene conocimiento y, por lo tanto, no existe una solución o parche disponible. El término "día cero" se refiere al hecho de que los desarrolladores tienen "cero días" para solucionar el problema una vez que la vulnerabilidad es descubierta y explotada.

1351.EXPLOIT KIT

Los kits de exploits (EK) son programas automatizados que utilizan los ciberdelincuentes para explotar sistemas o aplicaciones. Lo que hace que un kit de exploits sea muy peligroso es su capacidad para identificar a las víctimas mientras navegan por la web. Tras atacar las vulnerabilidades de una víctima potencial, los atacantes pueden descargar y ejecutar el malware que prefieran.

1352.PHISHING AVANZADO

Ataque sofisticado que utiliza ingeniería social y sitios web legítimos para lanzar ciberataques rápidos y a gran escala.

1353.SPEAR PHISHING

Ataque cibernético de ingeniería social que se dirige a un individuo o grupo específico de personas, a diferencia del phishing masivo que busca estafar a muchas personas a la vez. Los atacantes investigan a su objetivo para crear un mensaje falso y personalizado que parezca provenir de una fuente de confianza, con el fin de engañarlos para que revelen información confidencial, descarguen malware o realicen transacciones financieras.

1354.WHALING ATTACK

Es una forma de ciberataque de suplantación de identidad dirigido a ejecutivos de alto perfil y personas con autoridad en una organización, con el objetivo de engañarlos para que realicen transferencias de dinero, divulguen información confidencial o autoricen el acceso a datos sensibles, mediante correos electrónicos o mensajes altamente personalizados y convincentes.

1355.VISHING

Es una estafa de ingeniería social que combina el "phishing" con el uso de la voz, utilizando llamadas telefónicas fraudulentas para engañar a las víctimas y obtener información personal y financiera confidencial, como contraseñas, números de tarjeta de crédito o datos bancarios. Los delincuentes se hacen pasar por organizaciones o personas de confianza, como bancos o agencias gubernamentales, y emplean tácticas de manipulación, como la urgencia o el miedo, para presionar a la víctima a revelar sus datos.

1356.SMISHING

Es un tipo de ciberataque que combina "SMS" y "phishing", donde los atacantes envían mensajes de texto (SMS) fraudulentos para engañar a las víctimas, haciéndose pasar por fuentes confiables como bancos o empresas, con el objetivo de obtener información personal sensible como contraseñas o datos bancarios, o para que hagan clic en enlaces maliciosos que instalen malware o los dirijan a sitios web falsos.

1357.IA EMOCIONAL

Es una rama de la informática que dota a los sistemas de la capacidad de reconocer, interpretar, simular y responder a las emociones humanas.

1358.KEYLOGGER

Es un tipo de software o hardware que registra en secreto cada tecla pulsada en un dispositivo, como una computadora o un teléfono móvil, para robar información confidencial como contraseñas, datos bancarios o información personal. Pueden ser instalados por cibercriminales a través de descargas de malware o correos electrónicos de phishing, y la información capturada es enviada a un atacante remoto.

1359.ROOTKIT

Es un software malicioso que busca obtener y mantener acceso no autorizado a un sistema, al mismo tiempo que oculta su presencia y sus actividades ilícitas. Utiliza "kits" de herramientas de software para obtener "root" (acceso de administrador) en sistemas operativos, lo que le permite manipular archivos, ocultar procesos y datos, y evadir la detección por parte de programas de seguridad. Los atacantes pueden instalar rootkits mediante ataques de ingeniería social, explotando vulnerabilidades o, en algunos casos, a través de otros malware como virus.

1360.BOTNET

Red de dispositivos de cómputo (como computadoras, teléfonos, dispositivos IoT, etc.) que han sido infectados con malware y que son controlados de forma remota por un atacante, llamado "bot maestro" o "bot herder".

1361.DDOS (DISTRIBUTED DENIAL OF SERVICE)

Es un ciberataque que busca inutilizar un sitio web, servidor o recurso de red sobrecargando con una enorme cantidad de tráfico malicioso o solicitudes.

1362.RANSOMWARE

En el ransomware de extorsión doble, los atacantes no sólo cifran archivos, sino que también roban datos confidenciales y amenazan con liberarlos públicamente si no se paga el rescate. Con la destrucción de datos, se amenaza con destruir los datos de la víctima si esta no paga el rescate.

1363.SPYWARE

Es un tipo de software malicioso que se instala en un dispositivo (como una computadora o teléfono) sin el consentimiento del usuario para recopilar información privada y enviarla a terceros.

1364.ADWARE

Software diseñado para mostrarte publicidad no deseada, a menudo en forma de ventanas emergentes o anuncios en tu navegador web, con el objetivo de generar ingresos para su desarrollador.

1365.MALWARE POLIMÓRFICO

Es un software malicioso que cambia su código y firma cada vez que se replica o infecta un nuevo sistema, utilizando un cifrado para modificar su apariencia mientras mantiene intactas sus funciones principales. Esto le permite evadir los sistemas de detección antivirus que buscan patrones de código específicos, haciendo que el malware sea mucho más difícil de identificar y neutralizar.

1366.MALWARE FILELESS

(O malware sin archivos) es una forma de ataque cibernético que opera directamente en la memoria RAM del sistema en lugar de instalar archivos en el disco duro. Utiliza herramientas legítimas del sistema operativo, como PowerShell o WMI, para ejecutar código malicioso, lo que permite evadir los antivirus tradicionales y dificulta su detección, ya que no deja rastros en el disco duro.

1367.MALWARE EN FIRMWARE

Código malicioso inyectado en el firmware de un dispositivo, lo que le permite ejecutarse a un nivel muy bajo, incluso antes de que se inicie el sistema operativo. Esto dificulta su detección y eliminación, y es particularmente peligroso porque puede persistir después de reiniciar el sistema o reinstalar el sistema operativo, permitiendo a los atacantes robar datos, alterar el funcionamiento del sistema o crear puertas traseras.

1368.ADVANCED PERSISTENT THREAT (APT)

Código malicioso inyectado en el firmware de un dispositivo, lo que le permite ejecutarse a un nivel muy bajo, incluso antes de que se inicie el sistema operativo. Esto dificulta su detección y eliminación, y es particularmente peligroso porque puede persistir después de reiniciar el sistema o reinstalar el sistema operativo, permitiendo a los atacantes robar datos, alterar el funcionamiento del sistema o crear puertas traseras.

1369.KILL CHAIN

Es un modelo de ciberseguridad desarrollado por Lockheed Martin que descompone un ataque cibernético en sus etapas fundamentales, permitiendo a los equipos de seguridad identificar y detener el ataque antes de que logre su objetivo final.

1370.MITRE ATT&CK

Marco de ciberseguridad desarrollado por MITRE Corporation que documenta las tácticas, técnicas y procedimientos (TTP) que los adversarios usan durante los ciberataques, organizándose en matrices de datos.

1371.OWASP TOP 10

Es una lista estandarizada y ampliamente reconocida que identifica los 10 riesgos más críticos de seguridad en aplicaciones web, basándose en datos y en el consenso de expertos de seguridad a nivel mundial.

1372.INYECCION SQL

Es una vulnerabilidad de seguridad informática que permite a un atacante insertar código SQL malicioso en una aplicación web para manipular o acceder a una base de datos subyacente. El atacante abusa de una falta de validación en las entradas de la aplicación para ejecutar comandos SQL no autorizados, lo que puede permitir extraer, modificar o eliminar datos.

1373.CROSS-SITE SCRIPTING (XSS)

Es un tipo de ataque informático que permite a un atacante inyectar código malicioso (generalmente JavaScript) en un sitio web de confianza, para que luego se ejecute en el navegador de otros usuarios que visiten esa página. Esto puede provocar el robo de datos sensibles como las cookies de sesión, el secuestro de la identidad del usuario o la suplantación de identidad.

1374.CROSS-SITE REQUEST FORGERY (CSRF)

Es un ataque cibernético que engaña a un usuario autenticado en una aplicación web para que ejecute acciones no deseadas. Un atacante logra que el usuario interactúe con un sitio web malicioso, el cual contiene una solicitud que se envía automáticamente a una aplicación vulnerable a la que el usuario ya ha iniciado sesión. Esto permite al atacante realizar operaciones no autorizadas, como transferir fondos o alterar información, a nombre del usuario.

1375.PRIVILEGE ESCALATION

Técnica de ciberataque en la que un actor de amenazas altera o eleva sus permisos en un sistema objetivo, por ejemplo, pasando de una cuenta de usuario básica con menores privilegios a una cuenta de administrador de nivel superior.

1376.LATERAL MOVEMENT

Es una técnica que usan los ciberdelincuentes para navegar y propagarse dentro de una red comprometida después de obtener un acceso inicial, moviéndose sigilosamente de un dispositivo a otro para encontrar vulnerabilidades, escalar privilegios (obtener más permisos) y alcanzar su objetivo final, como datos sensibles, sistemas críticos o desplegar ransomware.

1377.EXFILTRACIÓN DE DATOS

Transferencia no autorizada y deliberada de información confidencial o sensible desde un sistema informático o una red a una ubicación externa controlada por un atacante.

1378.HONEYPOT

Es una trampa virtual, un sistema o recurso informático intencionalmente vulnerable y aislado, que sirve como señuelo para atraer a ciberatacantes.

1379.HONEYNET

Red de computadoras que utiliza múltiples honeypots (sistemas señuelo) para atraer y estudiar a los ciberatacantes.

1380.SANDBOXING MALWARE

Técnica de ciberseguridad que consiste en ejecutar archivos o código sospechoso en un entorno virtual aislado y controlado, llamado sandbox, para observar su comportamiento sin exponer la red o los sistemas reales a riesgos.

1381.SOC (SECURITY OPERATIONS CENTER)

Es un equipo y una función centralizada de una organización que se dedica a prevenir, detectar, analizar y responder a amenazas de ciberseguridad en tiempo real, para mejorar la seguridad general de la organización.

1382 SIEM AVANZADO

Plataforma de ciberseguridad que utiliza aprendizaje automático (IA) y análisis de datos avanzado para recopilar, correlacionar y analizar en tiempo real volúmenes masivos de registros y eventos de toda una organización (servidores, redes, dispositivos, la nube, etc.) con el objetivo de detectar anomalías, identificar amenazas de seguridad sofisticadas, priorizar alertas, investigar incidentes y automatizar respuestas de manera más precisa y eficiente que un sistema SIEM tradicional.

1383 IDS/IPS DE NUEVA GENERACIÓN

Es un sistema avanzado de ciberseguridad que va más allá de la detección y prevención básicas de intrusiones, integrando capacidades como inteligencia artificial (IA) y aprendizaje automático para identificar amenazas complejas y desconocidas.

1384 ENDPOINT DETECTION AND RESPONSE (EDR)

Es una tecnología de ciberseguridad que monitorea continuamente los puntos finales (como computadoras, laptops y móviles) para detectar amenazas avanzadas y permite a los equipos de seguridad responder automáticamente o manualmente a ellas para mitigarlas.

1385 XDR (EXTENDED DETECTION AND RESPONSE)

Plataforma unificada de ciberseguridad que recopila y correlaciona datos de telemetría de múltiples capas de seguridad (como endpoints, correo electrónico, nube, redes) utilizando inteligencia artificial y automatización para detectar, investigar y responder a amenazas avanzadas de manera más rápida y eficiente.

1386 SOAR (SECURITY ORCHESTRATION, AUTOMATION AND RESPONSE)

Conjunto de servicios y herramientas que unifican herramientas de seguridad, automatizan tareas repetitivas y coordinan la respuesta a incidentes cibernéticos mediante el uso de planes de acción predefinidos (playbooks).

1387 SEGURIDAD ZERO TRUST

Es un enfoque de ciberseguridad que parte del principio de "nunca confiar, verificar siempre". En lugar de asumir que cualquier usuario o dispositivo dentro de la red es seguro, exige que se verifique la identidad de todos y cada uno de los accesos a los recursos, otorgando el privilegio mínimo necesario y segmentando la red en zonas más pequeñas para minimizar el riesgo.

1388 AUTENTICACIÓN MULTIFACTOR ADAPTATIVA

Es un sistema de seguridad que utiliza información contextual para decidir qué factores de autenticación se requieren, ofreciendo más seguridad cuando se detectan actividades de riesgo y menos pasos para el usuario en situaciones normales, mejorando así el equilibrio entre seguridad y experiencia del usuario.

1389 PASSWORDLESS AUTHENTICATION

Método de seguridad que permite a los usuarios acceder a sistemas o aplicaciones sin necesidad de introducir una contraseña tradicional, utilizando en su lugar factores de autenticación alternativos como la biometría (huella dactilar, reconocimiento facial), dispositivos (llaves de seguridad físicas, teléfonos con apps de autenticación) o códigos de un solo uso (enviados por SMS o correo electrónico).

1390 ANÁLISIS FORENSE DIGITAL

Llamado informática forense o ciber forense, es la disciplina que aplica técnicas científicas para la adquisición, preservación, análisis y presentación de evidencia digital obtenida de dispositivos electrónicos con el fin de recopilar información que ayude en investigaciones legales y judiciales.

1391 INTERNET DE LAS COSAS (IOT)

Es una red de objetos físicos (dispositivos) con sensores, software y otras tecnologías, que se conectan e intercambian datos entre sí y con otros sistemas a través de internet, sin intervención humana directa.

1392 EDGE IOT

Computación de borde (edge computing) aplicada al Internet de las Cosas (IoT), donde el procesamiento y análisis de datos se realiza en o cerca de los dispositivos IoT, en lugar de enviarlos a una nube centralizada.

1393 FOG COMPUTING

Modelo de informática distribuida que acerca los recursos de procesamiento, almacenamiento y comunicación de la nube a la fuente donde se generan los datos, es decir, al borde de la red.

1394 IOT GATEWAY

Es un dispositivo o plataforma que funciona como un centro para conectar dispositivos y sensores del Internet de las Cosas (IoT) con la nube o servidores centrales, gestionando la comunicación, el procesamiento de datos y la seguridad en el "borde" de la red.

1395 IOT HUB

Es un servicio en la nube, como Azure IoT Hub, que sirve como un punto central para conectar, monitorear y administrar miles de millones de dispositivos IoT (Internet de las Cosas).

1396 MQTT

Protocolo de mensajería ligero de publicación-suscripción (Pub/Sub) diseñado para la comunicación entre dispositivos, especialmente en el ámbito del Internet de las Cosas (IoT) y de máquina a máquina (M2M).

1397 COAP

Protocolo de Aplicación Restringida (Constrained Application Protocol), un protocolo ligero de capa de aplicación diseñado para el Internet de las Cosas (IoT). Funciona sobre UDP para permitir que dispositivos con recursos limitados, como sensores de bajo consumo, se comuniquen eficientemente en redes con bajo ancho de banda y potencia de procesamiento, utilizando un modelo de solicitud-respuesta similar a HTTP pero más eficiente.

1398 LWM2M

Es un protocolo estándar desarrollado por la Open Mobile Alliance (OMA) para la gestión remota de dispositivos del Internet de las Cosas (IoT) con recursos limitados, como baja potencia, memoria y capacidad de procesamiento.

1399 THREAD

Secuencia más pequeña de instrucciones programadas que un sistema operativo puede gestionar de forma independiente, permitiendo que un programa realice múltiples tareas simultáneamente dentro de un mismo proceso, compartiendo así el mismo espacio de memoria y recursos.

1400 MATTER (PROTOCOLO IOT)

Estándar de conectividad abierto y unificado para la Internet de las Cosas (IoT), enfocado en hogares inteligentes, que permite que dispositivos de diferentes fabricantes se comuniquen y funcionen juntos sin problemas, independientemente del ecosistema que utilice el usuario.

1401 SMART HOME

Es una vivienda que integra tecnologías de la Internet de las Cosas (IoT) para conectar y automatizar diversos sistemas y dispositivos, como la iluminación, la seguridad, el aire acondicionado y los electrodomésticos.

1402 SMART CITY

Utiliza la informática y las Tecnologías de la Información y la Comunicación (TIC) para mejorar la calidad de vida de sus ciudadanos, la eficiencia de los servicios urbanos y la sostenibilidad de la ciudad.

1403 SMART GRID

Es un sistema de distribución de electricidad que integra tecnologías de información y comunicación digital para gestionar de forma dinámica la oferta y la demanda de energía en tiempo real.

1404 WEARABLES IOT

(Dispositivos vestibles del Internet de las Cosas) son aparatos electrónicos portátiles, como relojes inteligentes o pulseras de actividad, que incorporan sensores para recoger y transmitir datos a través de una red, permitiendo así monitorear aspectos del usuario o del entorno y conectar con otros dispositivos para ofrecer funciones más avanzadas.

1405 SENSORES IOT

Dispositivo físico que detecta y mide cambios en su entorno (como temperatura, movimiento o luz), y luego convierte esa información en datos que se transmiten a través de internet a otros sistemas o a la nube para su análisis y uso.

1406 ACTUADORES IOT

Dispositivo que convierte una señal eléctrica en una acción física en el mundo real, permitiendo a los sistemas del Internet de las Cosas (IoT) interactuar y manipular su entorno. A diferencia de los sensores, que captan datos del entorno y los convierten en señales eléctricas, los actuadores realizan el movimiento o cambio físico en respuesta a esas señales, conectando así el mundo digital con el físico.

1407 TELEMÁTICA IOT

Son campos interconectados que se apoyan entre sí: la Telemática combina la informática y las telecomunicaciones para el intercambio remoto de datos; el IoT es una aplicación de la informática que conecta objetos físicos a internet para que recopilen e intercambien datos; y la Informática es la base fundamental, proporcionando la tecnología (software, hardware, redes) necesaria para que la telemática y el IoT funcionen y se desarrollen.

1408 IOT EN SALUD

Integración de dispositivos conectados a Internet (como sensores, wearables y equipos médicos) para recopilar, transmitir y analizar datos de salud en tiempo real, permitiendo un mejor monitoreo, diagnóstico y tratamiento de pacientes, así como una gestión más eficiente de la atención médica.

1409 IOT EN TRANSPORTE

Aplicación de Internet de las Cosas para conectar sensores, vehículos y sistemas, permitiendo la recopilación y análisis de datos en tiempo real para optimizar rutas, mejorar la seguridad, gestionar el mantenimiento predictivo, mejorar la experiencia del cliente y aumentar la eficiencia en la cadena de suministro, todo a través de la comunicación entre dispositivos inteligentes y sistemas basados en Internet.

1410 IOT EN AGRICULTURA

Aplicación de sensores, dispositivos conectados y análisis de datos para optimizar los procesos de cultivo, aumentando la eficiencia y reduciendo el desperdicio de recursos.

1411 IOT INDUSTRIAL (IIOT)

Aplicación del Internet de las Cosas en el ámbito industrial, conectando maquinaria, sensores y sistemas para recopilar, analizar y actuar sobre datos en tiempo real.

1412 INDUSTRIA 4.0

Integración de tecnologías digitales avanzadas en la fabricación y los procesos industriales, combinando el mundo físico y el digital para crear "fábricas inteligentes" autónomas y eficientes.

1413 GEMELO DIGITAL

Representación virtual dinámica de un objeto, sistema o proceso del mundo real, que se actualiza en tiempo real con datos de sensores para simular y predecir su comportamiento y rendimiento.

1414 MANTENIMIENTO PREDICTIVO IOT

Estrategia que usa sensores IoT para recopilar datos en tiempo real de máquinas y equipos, los cuales son analizados por software y algoritmos de inteligencia artificial (IA) para predecir fallas antes de que ocurran.

1415 SEGURIDAD IOT

1416 BOTNET MIRAI

1417 IOT FORENSE

1418 ARQUITECTURA IOT

Marco técnico y estructural que define cómo los distintos componentes del Internet de las Cosas (IoT)—como sensores, dispositivos, redes, software y la nube—interactúan para recopilar, transmitir, procesar y analizar datos, permitiendo la comunicación y el funcionamiento inteligente de un ecosistema de IoT.

1419 EDGE AI

o "IA en el perímetro", se refiere a la combinación de computación edge e inteligencia artificial para ejecutar tareas de machine learning directamente en dispositivos edge interconectados.

1420 TINY ML

1422 SIGFOX

Infraestructura y aplicaciones en la nube que permiten la recopilación, almacenamiento, análisis e inteligencia de los datos generados por los dispositivos del Internet de las Cosas (IoT), facilitando la comunicación y el monitoreo remoto de objetos físicos.

1426 AWS IOT HUB

1430 OPENHAB

Plataforma de automatización del hogar (domótica) de código abierto diseñada para integrar y controlar una amplia variedad de dispositivos y sistemas inteligentes, sin importar su marca o protocolo de comunicación.

1431 HOME ASSISTANT

1432 DOMÓTICA AVANZADA

Integración inteligente y automatizada de tecnologías en el hogar o edificio para gestionar y controlar sistemas como la climatización, iluminación, seguridad y electrodomésticos, a menudo utilizando el Internet de las Cosas (IoT) y la inteligencia artificial para ofrecer mayor confort, seguridad, eficiencia energética y bienestar a los usuarios.

Automóvil equipado con tecnología que le permite acceder a Internet y comunicarse con su entorno, intercambiando información con otros vehículos (V2V), infraestructura (V2I), y la nube para servicios como GPS, actualizaciones de software, información del tráfico, y asistentes de conducción, mejorando así la seguridad y la experiencia del conductor.

1434 V2X (VEHICLE-TO-EVERYTHING)

Tecnología de comunicación inalámbrica que permite a los vehículos intercambiar datos en tiempo real con todo su entorno.

1435 IOT EN ROBÓTICA

Conexión de robots y dispositivos robóticos a Internet mediante sensores, software y otras tecnologías, lo que les permite recopilar y compartir datos en tiempo real.

1436 IOT EN ENERGÍA RENOVABLE

Consiste en conectar dispositivos como sensores y equipos de generación de energía a internet para recopilar, transmitir y analizar datos en tiempo real. Esta tecnología permite la monitorización y el control remoto de las instalaciones, la automatización de procesos, la optimización de la generación y el consumo energético, la gestión eficiente de las redes inteligentes (Smart Grids) y la integración de fuentes de energía limpia, lo que resulta en una mayor eficiencia, reducción de costos y una menor dependencia de combustibles fósiles.

1437 ESTANDARIZACIÓN IOT

Establecimiento de protocolos, formatos de datos y guías de seguridad universales que permiten la interoperabilidad, la comunicación y el intercambio de datos seguros entre diversos dispositivos y sistemas en el Internet de las Cosas (IoT).

1438 INTEROPERABILIDAD IOT

Capacidad de diferentes dispositivos y sistemas del Internet de las Cosas de comunicarse, intercambiar e interpretar datos de manera fluida y efectiva.

1439 BLOCKCHAIN EN IOT

Es la fusión de la tecnología blockchain, una base de datos descentralizada, inmutable y segura, con el Internet de las Cosas (IoT), una red de dispositivos interconectados, para crear un registro a prueba de manipulaciones de transacciones y datos generados por los dispositivos IoT.

1440 ECONOMÍA DE DISPOSITIVOS IOT

Análisis y la gestión del valor económico que generan los dispositivos interconectados que recopilan e intercambian datos
